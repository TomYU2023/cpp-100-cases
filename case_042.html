新语法精讲

ANSI 转义序列

语法规则：终端中的颜色和样式通过 ANSI 转义序列 控制。格式：ESC[参数m，其中 ESC 字符写作 \033（八进制）或 \x1b（十六进制）。多个参数用分号分隔。强调：Windows 10 以下版本需调用 system("") 启用 ANSI 支持。
// ANSI 颜色/样式表
// 前景: 30黑 31红 32绿 33黄 34蓝 35品红 36青 37白
// 背景: 40黑 41红 42绿 43黄 44蓝 45品红 46青 47白
// 样式: 0重置 1粗体 2弱化 3斜体 4下划线 5闪烁 7反色 9删除线
cout << "\033[1;33;44m" << "粗体黄字蓝底" << "\033[0m";
cout << "\033[4;31m" << "红色下划线" << "\033[0m";

std::ostringstream 字符串流

语法规则：ostringstream 是一个输出字符串流，行为类似 cout 但输出到内存中的字符串。通过 str() 方法获取累积的字符串内容。这在需要动态构建字符串时非常有用，避免了频繁的字符串拼接开销。
ostringstream oss;
oss << "Hello " << 42 << " World";
string result = oss.str();  // "Hello 42 World"
// 清空重用
oss.str("");   // 清空内容
oss.clear();  // 清除错误状态

方法链式调用（Fluent API / Builder 模式）

语法规则：让成员函数返回 *this（自身引用），实现 obj.func1().func2().func3() 的链式写法。这是 Builder 模式的核心技巧，常见于 cout（cout << a << b << c）和现代 C++ 库。
class Builder {
    int x, y;
public:
    Builder& setX(int v) { x = v; return *this; }
    Builder& setY(int v) { y = v; return *this; }
};
Builder().setX(1).setY(2);  // 链式调用

举一反三

  • 跨平台颜色库（fmt / rang）：
    fmt::print(fg(fmt::color::red), "error!\n");
// 或
cout << rang::fg::red << "error!" << rang::style::reset;

    fmt 库（C++20 中成为 std::format 的基础）和 rang 库提供了跨平台的终端颜色支持，自动处理 Windows/Linux/macOS 差异。

  • Windows Console API 原生着色：
    HANDLE h = GetStdHandle(STD_OUTPUT_HANDLE);
SetConsoleTextAttribute(h, FOREGROUND_RED | FOREGROUND_INTENSITY);
cout << "red text" << endl;

    Windows 提供原生 API 控制控制台颜色，不依赖 ANSI 码。适合需要精确控制或老旧 Windows 版本的场景。
