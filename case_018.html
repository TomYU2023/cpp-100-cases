案例 18：移动语义演练

通过实现一个带资源管理的 Buffer 类，深入理解移动构造和移动赋值的原理和用法。

核心知识点：移动构造、移动赋值、std::move、右值引用

完整代码（含逐行注释）

// 案例 18：移动语义演练
// 通过资源管理类理解移动构造和移动赋值
#include <iostream>
#include <vector>
#include <chrono>
using namespace std;

class Buffer {
private:
    size_t size;
    int* data;

public:
    explicit Buffer(size_t sz) : size(sz), data(new int[sz]) {
        cout << "构造: size=" << size << endl;
    }

    // 拷贝构造（深拷贝，昂贵！）
    Buffer(const Buffer& other) : size(other.size), data(new int[other.size]) {
        copy(other.data, other.data + size, data);
        cout << "拷贝构造: size=" << size << endl;
    }

    // 移动构造（窃取资源，O(1)！）
    Buffer(Buffer&& other) noexcept
        : size(other.size), data(other.data) {
        other.size = 0;      // 置空源对象
        other.data = nullptr;
        cout << "移动构造: size=" << size << endl;
    }

    // 移动赋值
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data;           // 释放当前资源
            size = other.size;
            data = other.data;       // 窃取
            other.size = 0;
            other.data = nullptr;
        }
        cout << "移动赋值: size=" << size << endl;
        return *this;
    }

    ~Buffer() {
        if (data) cout << "析构: size=" << size << endl;
        delete[] data;
    }
};

// 工厂函数：返回临时对象（触发移动语义）
Buffer createBuffer(size_t sz) {
    Buffer buf(sz);
    return buf; // NRVO / 移动构造
}

int main() {
    // 1. 从工厂函数接收（移动构造，不拷贝）
    Buffer b1 = createBuffer(1000);

    // 2. 显式使用 std::move 触发移动
    Buffer b2(500);
    Buffer b3 = move(b2); // b2 资源被窃取，b2 不再有效
    // 警告：此时不应再使用 b2（其 data 已被置空）

    // 3. vector 扩容时自动移动（如果移动构造 noexcept）
    vector<Buffer> buffers;
    buffers.reserve(3);
    buffers.push_back(Buffer(100));
    buffers.push_back(Buffer(200));
    buffers.push_back(Buffer(300));

    return 0;
}

语法规则：移动构造与右值引用

Buffer(Buffer&& other) noexcept
    : size(other.size), data(other.data) {
    other.data = nullptr; // 必须置空
}

举一反三：移动构造通过「窃取」资源避免深拷贝，参数是右值引用。必须将源对象置于可安全析构的状态（置空指针）。noexcept 对 vector 扩容至关重要。

语法规则：std::move 的作用

Buffer b2(100);
Buffer b3 = move(b2); // 将 b2 转为右值，触发移动构造

举一反三：move 本身不做任何移动，只是将左值强制转为右值引用，触发移动语义。move 后源对象处于「有效但未定义」状态，不应再使用。

语法规则：noexcept 与容器优化

// vector 扩容时优先使用移动（如果声明为 noexcept）
// 否则回退到拷贝（保证异常安全）

举一反三：移动操作应尽量声明 noexcept，使 STL 容器在 reallocation 时优先使用移动而非拷贝。这是 C++ 性能优化的关键细节。
