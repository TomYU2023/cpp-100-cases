案例 9：学生成绩管理系统(struct版)

使用结构体实现一个学生成绩管理系统，支持添加、查询、排序、统计功能。

核心知识点：struct 定义、vector 存储、sort 自定义排序

完整代码（含逐行注释）

// 案例 9：学生成绩管理系统 (struct 版)
// 使用结构体 + vector + lambda 排序实现管理功能
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>
#include <iomanip>
using namespace std;

// 学生结构体
struct Student {
    string name;
    int math;
    int english;
    int cs;

    // 计算总分
    int total() const { return math + english + cs; }
    // 计算平均分
    double avg() const { return total() / 3.0; }
};

// 打印单个学生信息
void printStudent(const Student& s) {
    cout << left << setw(10) << s.name
         << setw(8) << s.math
         << setw(8) << s.english
         << setw(8) << s.cs
         << setw(8) << s.total()
         << fixed << setprecision(1) << s.avg() << endl;
}

int main() {
    // 初始化学生数据
    vector<Student> students = {
        {"张三", 85, 92, 78},
        {"李四", 90, 88, 95},
        {"王五", 72, 85, 80},
        {"赵六", 95, 91, 88}
    };

    // 按总分降序排序（lambda 表达式自定义比较规则）
    sort(students.begin(), students.end(),
         [](const Student& a, const Student& b) {
             return a.total() > b.total(); // 总分高的排前面
         });

    // 打印表头
    cout << left << setw(10) << "姓名"
         << setw(8) << "数学"
         << setw(8) << "英语"
         << setw(8) << "C++"
         << setw(8) << "总分"
         << "平均分" << endl;
    cout << string(50, '-') << endl;

    // 打印所有学生
    for (const auto& s : students) {
        printStudent(s);
    }

    return 0;
}

struct 定义与成员函数

【规则】struct 定义自定义数据类型，可包含数据成员和成员函数。与 class 的区别仅在于默认访问权限：struct 默认 public，class 默认 private。适用于数据聚合和简单值对象。

【原理】C++ 的 struct 继承自 C 的 struct，但能力被大幅扩展——可以包含构造函数、析构函数、成员函数、继承、访问控制。语义约定上，struct 常用于「数据聚合」（POD/aggregate），class 常用于「封装对象」。但这只是约定——编译器对两者一视同仁。POD struct 的内存布局是连续的，可以直接 memcpy。

【常见陷阱】1. 在 struct 中使用 std::string 后试图用 memset 清零——string 内部有指针，memset 会把指针设为零，析构时崩溃；2. 以为 struct 不能有 private 成员——实际上可以，只是默认 public，显式写 private: 照样生效。

struct Student {
    string name;
    int score;
    int total() const { return score; }
};

【变体练习一】为案例 009 的 Student struct 添加成员函数 display()，将格式化输出的逻辑封装在 struct 内部。体现「数据 + 行为」的设计思想。

【变体练习二】将 Student struct 改为 class，显式声明 public/private，对比编译差异。然后定义一个 teacher class 继承自 person struct，观察 struct 的默认继承访问权限。

sort + lambda 自定义排序

【规则】std::sort(begin, end, lambda) 使用 lambda 表达式作为自定义比较器。Lambda 返回 true 表示第一个参数应排在第二个参数之前。适用于多条件排序、降序排列等。

【原理】sort 要求比较器满足「严格弱序」——即 irreflexive（a

【常见陷阱】1. Lambda 写成了 return a <= b（带等号）——这违反了严格弱序的 irreflexive 原则（a<=a 为 true，但要求必须为 false），sort 在遇到相等元素时可能越界；2. 忽视命名空间语法限制，将 lambda 表达式中捕获的 string 变量拼凑成多条件表达式时，少闭合了一个括号导致反直觉结果。

sort(v.begin(), v.end(),
     [](const T& a, const T& b) { return a.key > b.key; });

【变体练习一】按「总分降序，总分相同时按数学成绩降序，数学也相同时按姓名升序」的三级排序规则编写 lambda 比较器。提示：用 if-return 链而非 && || 组合。

【变体练习二】不用 lambda，改用函数对象（functor）实现相同的排序逻辑。定义一个 CompareByTotal 类，重载 operator()，传给 sort 的第三个参数。对比两种写法的代码量和可读性。

iomanip 格式化表格

【规则】使用 中的操纵符（setw、left、right、setfill、setprecision 等）控制输出格式，实现控制台表格的美观打印。适用于报表输出、数据展示等。

【原理】操纵符分两类：一次性（setw）和持久性（setfill、setprecision、left/right）。一次性操纵符在每次输出后自动重置，这是 C++ 流库防止格式污染的机制。底层原理是每个 ostream 对象有一个 fmtflags 状态字，操纵符修改这个状态字的不同位。理解这个机制让你能准确预测格式行为。

【常见陷阱】1. 使用 setw 设置每列宽度后忘记在列与列之间加空格分隔符，导致不同列内容粘连在一起；2. 中文和英文/数字混排时，中文占 2 倍宽度但 setw 按 char 计数，导致表格对不齐——需要根据中文字符数动态调整宽度或使用等宽中文字体。

#include <iomanip>
cout << left << setw(10) << name; // 左对齐，宽度 10

【变体练习一】在成绩管理系统中增加「平均分」列和「排名」列，用 setw 和 left/right 对齐所有列，输出一个完整美观的成绩总表。

【变体练习二】不使用 iomanip，改用 printf 风格的格式说明符（%s、%d、%-8s 等）实现相同的表格输出，对比两种风格在格式控制上的表达力差异。
