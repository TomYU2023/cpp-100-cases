案例 75：CMake 构建系统入门

展示从单文件到多目录库的完整 CMakeLists.txt 配置，详解 PRIVATE/PUBLIC/INTERFACE 依赖传递规则。

核心知识点：CMake 构建系统、target_link_libraries 传递控制、add_subdirectory、C++17 标准设置

完整代码（含逐行注释）

// 案例 75：CMake 构建系统入门
// CMake 是 C++ 生态的事实标准构建系统，用 CMakeLists.txt 描述项目结构。
// 本案例展示从单文件到多目录库的完整 CMake 配置。

/* 语法规则：CMake 核心语法
   CMake 命令不区分大小写（推荐小写），变量区分大小写。
   基本结构：
     cmake_minimum_required(VERSION 3.16)  # 最低版本要求
     project(MyProject VERSION 1.0 LANGUAGES CXX)  # 项目定义
     add_executable(app main.cpp)  # 生成可执行文件
     add_library(mylib STATIC src/lib.cpp)  # 生成库
     target_link_libraries(app PRIVATE mylib)  # 链接库

   变量：set(VAR value)、${VAR} 引用
   条件：if/elseif/else/endif
   循环：foreach/endforeach
   函数：function(name arg1 arg2)/endfunction

   示例：最简单的 CMakeLists.txt
     cmake_minimum_required(VERSION 3.16)
     project(Hello)
     add_executable(hello main.cpp)

   举一反三：CMake 还负责查找第三方库（find_package）、
   设置编译选项（target_compile_features）、安装规则（install）、
   打包（CPack）、测试（CTest）。掌握 CMake 是 C++ 工程师的硬门槛。
*/

// ============= CMakeLists.txt（项目根目录）=============
/*
cmake_minimum_required(VERSION 3.16)

# 项目信息
project(CppProject
    VERSION 1.0.0
    DESCRIPTION "C++ 项目演示"
    LANGUAGES CXX
)

# 设置 C++ 标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)     # 强制要求，不支持则报错
set(CMAKE_CXX_EXTENSIONS OFF)           # 禁用编译器扩展，保证可移植性

# 添加子目录（自动处理子目录的 CMakeLists.txt）
add_subdirectory(src)

# 可选：测试
option(BUILD_TESTS "构建测试" OFF)
if(BUILD_TESTS)
    enable_testing()
    add_subdirectory(tests)
endif()
*/

// ============= src/CMakeLists.txt =============
/*
# 静态库：核心功能
add_library(core STATIC
    core/utils.cpp
    core/logger.cpp
)

# 为目标设置包含目录（core/ 的头文件在 core/ 下）
target_include_directories(core PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})

# 可执行文件
add_executable(myapp main.cpp)

# 链接库：PRIVATE = 仅 myapp 使用，不传递给依赖者
target_link_libraries(myapp PRIVATE core)
*/

// ============= 构建命令 =============
// mkdir build && cd build
// cmake ..                    # 生成构建文件
// cmake --build .             # 编译
// cmake --build . --config Release  # 编译 Release 版
// cmake --install . --prefix ../out  # 安装

// ============= C++ 源码示例 =============
// main.cpp
#include <iostream>

int main() {
    std::cout << "案例 75：CMake 构建系统" << std::endl;
    std::cout << "=========================" << std::endl;
    std::cout << "CMake 核心概念：" << std::endl;
    std::cout << "  1. CMakeLists.txt — 项目描述文件" << std::endl;
    std::cout << "  2. add_executable — 构建可执行文件" << std::endl;
    std::cout << "  3. add_library — 构建库（STATIC/SHARED）" << std::endl;
    std::cout << "  4. target_link_libraries — 链接依赖" << std::endl;
    std::cout << "  5. PRIVATE/PUBLIC/INTERFACE — 依赖传递控制" << std::endl;
    std::cout << "\n依赖传递规则：" << std::endl;
    std::cout << "  PRIVATE   — 仅自己用，不传递给下游" << std::endl;
    std::cout << "  PUBLIC    — 自己用 + 下游也能用" << std::endl;
    std::cout << "  INTERFACE — 自己不用，下游用（纯头文件库）" << std::endl;
    return 0;
}

语法规则：CMake PRIVATE/PUBLIC/INTERFACE 传递控制

# A → B → C（链接依赖链）
# target_link_libraries(B PRIVATE A)
#   → C 看不到 A 的头文件和符号
# target_link_libraries(B PUBLIC A)
#   → C 可以看到 A
# target_link_libraries(B INTERFACE A)
#   → B 自己不用 A，但 C 能看（常用于 header-only lib）

💡 举一反三：这是 CMake 最核心的设计概念。PUBLIC 用于 API 暴露（如你的库头文件引用了第三方库），PRIVATE 用于内部实现细节。

语法规则：target_include_directories 包含路径

target_include_directories(mylib PUBLIC include/)
# PUBLIC: 使用 mylib 的目标也能访问 include/ 下的头文件
# PRIVATE: 只有 mylib 自己编译时能用

💡 举一反三：不要再用 include_directories() 全局设置！它是 CMake 2.x 的遗留物，污染所有目标。
