新语法精讲

回调函数与 void* 上下文参数

语法规则：C 风格的 API 常用回调 + void* 传递用户数据。void* 是通用指针，需要 static_cast 转回实际类型。Lambda 只要无捕获就可以转为函数指针（有捕获时不行）。
// C API 的回调模式
typedef int (*Callback)(void* userdata, int col, char** vals);

// 我们的数据
vector results;

// Lambda 作为回调
int rc = db_exec(sql, [](void* p, int n, char** v, char**) {
    auto* rows = static_cast*>(p);
    // ... 填充 rows
    return 0;
}, &results);

sqlite3_free 与 C 内存管理

语法规则：SQLite 用 sqlite3_malloc 分配内存，错误信息也由它分配。必须用 sqlite3_free 释放，不能用 C++ 的 delete 或 free。这是 C 库集成时常见的注意事项。
char* errMsg = nullptr;
int rc = sqlite3_exec(db, sql, nullptr, nullptr, &errMsg);
if (rc != SQLITE_OK) {
    cerr << errMsg;        // 使用错误信息
    sqlite3_free(errMsg);  // 释放！
}

举一反三

  • 使用 Prepared Statement 防 SQL 注入：
    sqlite3_stmt* stmt;
sqlite3_prepare_v2(db, "SELECT * FROM users WHERE name = ?", -1, &stmt, 0);
sqlite3_bind_text(stmt, 1, name.c_str(), -1, SQLITE_STATIC);
while (sqlite3_step(stmt) == SQLITE_ROW) {
    int age = sqlite3_column_int(stmt, 2);
}

    直接用 sqlite3_exec 拼接 SQL 字符串存在 SQL 注入风险。生产代码必须使用 Prepared Statement（预编译语句）。

  • ORM 库简化数据库操作：
    C++ 生态中有 sqlite_orm（header-only，使用模板元编程实现 ORM）和 SOCI 等库，可以将 SQL 操作封装为 C++ 对象操作，极大简化代码。

    #include using namespace sqlite_orm; auto storage = make_storage("db.sqlite", make_table("users", make_column("id", &User::id, primary_key().autoincrement()), make_column("name", &User::name))); auto users = storage.get_all();
