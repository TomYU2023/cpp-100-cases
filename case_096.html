案例 96：简易容器

用 clone() + Linux Namespace 实现容器隔离：PID 命名空间、UTS 命名空间、独立文件系统，理解 Docker 底层原理。

核心知识点：clone() 系统调用、Linux Namespace、位掩码标志、errno/strerror、容器隔离原理

完整代码（含逐行注释）

// 案例 96：简易容器 —— 理解 Docker 核心原理
// Docker 依赖 Linux Namespace（隔离）和 Cgroups（限制）。
// 本案例用 clone() 系统调用创建隔离的进程空间。
// 需在 Linux 下编译运行: g++ -std=c++17 container.cpp -o container

#define _GNU_SOURCE
#include <iostream>
#include <string>
#include <cstring>
#include <sched.h>      // clone()
#include <sys/wait.h>   // waitpid()
#include <sys/mount.h>  // mount()
#include <unistd.h>
#include <signal.h>

const int STACK_SIZE = 1024 * 1024;  // 子进程栈 1MB
char childStack[STACK_SIZE];

// 子进程入口（容器内执行的程序）
int childFunc(void* arg) {
    // 1. 设置 hostname（UTS namespace 隔离）
    sethostname("my-container", 11);

    std::cout << "=== 容器内环境 ===" << std::endl;
    std::cout << "PID: " << getpid() << std::endl;
    std::cout << "PPID: " << getppid() << std::endl;

    // 容器内可以看到独立的 PID 命名空间
    char hostname[256];
    gethostname(hostname, sizeof(hostname));
    std::cout << "Hostname: " << hostname << std::endl;

    std::cout << "容器启动成功！（输入 exit 退出）" << std::endl;
    return 0;
}

int main() {
    std::cout << "=== 简易容器模拟 ===" << std::endl;
    std::cout << "宿主机 PID: " << getpid() << std::endl;

    // Namespace 标志位（按位或组合）
    // CLONE_NEWPID  — PID 命名空间（容器内 PID=1）
    // CLONE_NEWUTS  — UTS 命名空间（独立 hostname）
    // CLONE_NEWNS   — Mount 命名空间（独立文件系统挂载）
    int flags = CLONE_NEWPID | CLONE_NEWUTS | CLONE_NEWNS | SIGCHLD;

    std::cout << "创建容器进程..." << std::endl;

    // clone() 创建子进程并加入新命名空间
    pid_t childPid = clone(childFunc, childStack + STACK_SIZE, flags, nullptr);

    if (childPid < 0) {
        std::cerr << "clone() 失败: " << strerror(errno) << std::endl;
        std::cerr << "提示: 需要 root 权限运行此程序。" << std::endl;
        return 1;
    }

    std::cout << "容器进程 PID: " << childPid << std::endl;
    std::cout << "等待容器进程退出..." << std::endl;

    int status;
    waitpid(childPid, &status, 0);
    std::cout << "容器已退出 (状态码: " << WEXITSTATUS(status) << ")" << std::endl;
    return 0;
}

语法规则：clone() 系统调用与 Linux Namespace

#define _GNU_SOURCE
#include <sched.h>
char stack[1024*1024];
int flags = CLONE_NEWPID | CLONE_NEWUTS | CLONE_NEWNS | SIGCHLD;
pid_t pid = clone(childFunc, stack + sizeof(stack), flags, arg);

举一反三：clone() 是 Linux 创建进程/线程的底层系统调用（fork() 和 pthread_create 都基于它）。flags 指定创建哪些独立命名空间：CLONE_NEWPID（容器内 PID 从 1 开始）、CLONE_NEWUTS（独立 hostname）、CLONE_NEWNS（独立挂载点）。Docker 就是组合这些标志位实现容器隔离。

语法规则：位掩码标志组合（Bitmask Flags）

// 每个标志是 2 的不同次幂，按位或组合
const int FLAG_A = 1 << 0;  // 0x01
const int FLAG_B = 1 << 1;  // 0x02
int flags = FLAG_A | FLAG_B;        // 组合
if (flags & FLAG_A) { /* A 已设置 */ }
flags &= ~FLAG_A;                   // 清除 A

举一反三：位掩码是 C 语言和 Linux 系统调用的经典模式。每个标志位占一个 bit，按位或(|)组合，按位与(&)检测，按位取反再与清除。相比传多个 bool 参数，位掩码更高效且可扩展。

语法规则：errno 和 strerror 错误处理

#include <cerrno>
#include <cstring>
if (clone(...) < 0) {
    std::cerr << "失败: " << strerror(errno) << std::endl;
}

举一反三：系统调用失败时设置全局变量 errno（线程局部存储），strerror() 将错误码转为人类可读的描述（如 'Permission denied'）。在多线程环境中每个线程有独立的 errno。
