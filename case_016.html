案例 16：异常处理实战

实现一个除法函数并处理除零异常，练习 try-catch 和栈展开机制。

核心知识点：try-catch、throw、栈展开、noexcept

完整代码（含逐行注释）

// 案例 16：异常处理实战
// 练习 try-catch、栈展开和资源安全
#include <iostream>
#include <stdexcept>
#include <string>
using namespace std;

// 自定义异常类
class DivByZeroException : public runtime_error {
public:
    DivByZeroException() : runtime_error("除数不能为零！") {}
};

// 安全的除法函数
double safeDivide(double a, double b) {
    if (b == 0) {
        throw DivByZeroException(); // 抛出自定义异常
    }
    return a / b;
}

// 资源管理类：演示栈展开时自动调用析构
class Resource {
    string name;
public:
    Resource(const string& n) : name(n) {
        cout << "Resource [" << name << "] 已获取" << endl;
    }
    ~Resource() {
        cout << "Resource [" << name << "] 已释放" << endl;
    }
};

int main() {
    double x = 10, y = 0;

    try {
        Resource res("文件句柄"); // 栈对象，异常时会自动析构
        cout << "开始计算 " << x << " / " << y << " ..." << endl;
        double result = safeDivide(x, y);
        cout << "结果 = " << result << endl;
    }
    catch (const DivByZeroException& e) {
        cerr << "自定义异常: " << e.what() << endl;
    }
    catch (const exception& e) {
        cerr << "标准异常: " << e.what() << endl;
    }
    catch (...) {
        cerr << "未知异常！" << endl;
    }

    cout << "程序继续执行..." << endl;
    return 0;
}

语法规则：try-catch 异常捕获

try {
    riskyCode();
} catch (const MyException& e) {
    cerr << e.what();
} catch (...) {
    // 兜底
}

举一反三：catch 块按声明顺序匹配，派生类异常应先于基类。catch(...) 必须放在最后。异常未被捕获时程序调用 std::terminate 终止。

语法规则：栈展开与 RAII

// 异常抛出时，从 throw 点向上搜索 catch
// 期间所有局部对象的析构函数被调用
{
    Resource r; // 构造
    throw exception(); // r 自动析构
}

举一反三：这是 C++ 异常安全的核心机制。确保所有资源用 RAII 管理（智能指针、容器等），这样异常发生时资源不会泄漏。

语法规则：自定义异常类

class MyException : public runtime_error {
public:
    MyException(const string& msg) : runtime_error(msg) {}
};

举一反三：继承 std::exception 或其子类。what() 返回 const char*，建议用 runtime_error 或 logic_error 做基类。命名以 Exception 或 Error 结尾。
