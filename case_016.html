案例 16：异常处理实战

完整代码（带逐行注释）

1 #include <iostream>
2 #include <fstream> 文件流
3 #include <stdexcept> 标准异常
4 using namespace std;
5
6 int safeDivide(int a, int b) {
7 if (b == 0) {
8 throw runtime_error("除数不能为零！"); throw 抛出异常对象
9 }
10 return a / b; 只有 b != 0 才会执行到这里
11 }
12
13 void readFile(const string& path) {
14 ifstream file(path); 尝试打开文件
15 if (!file.is_open()) { 打开失败
16 throw runtime_error("无法打开文件: " + path); 异常的 what() 方法返回错误信息
17 }
18 cout << "文件打开成功！" << endl;
19 file.close();
20 }
21
22 int main() {
23
24 try { try 块：可能抛出异常的代码
25 cout << safeDivide(10, 2) << endl; 正常计算
26 cout << safeDivide(10, 0) << endl; 抛出异常，跳转到 catch
27 cout << "这行不会执行" << endl; 异常后面代码被跳过
28 } catch (const runtime_error& e) { 捕获 runtime_error 异常
29 cerr << "运行时错误: " << e.what() << endl; e.what() 获取错误信息
30 } catch (const exception& e) { 捕获所有标准异常的基类
31 cerr << "其他异常: " << e.what() << endl;
32 }
33
34 try {
35 readFile("不存在的文件.txt"); 必定抛出异常
36 } catch (const runtime_error& e) { 可以有多层 try-catch
37 cerr << e.what() << endl;
38 }
39
40 cout << "程序继续运行" << endl; 异常被捕获后程序正常运行
41 return 0;
42 }

新语法讲解

异常处理的完整语法

try { ... } catch (异常类型& e) { ... }。throw 抛出的异常沿调用栈向上传递，直到被 catch 捕获。如果始终未被 catch，程序终止。
try {
    throw 42;          // 抛 int
} catch (int x) {
    cout << x;         // 捕获
} catch (...) {       // 捕获所有类型
    cout << "unknown";
}

标准异常类层次

exception 是所有标准异常的基类。重要子类：runtime_error（运行时错误）、logic_error（逻辑错误）、out_of_range、invalid_argument。
#include <stdexcept>
throw invalid_argument("age must be >= 0");
throw out_of_range("index 5 out of range for array size 3");

举一反三

自定义异常类

代码：class MyError : public exception { const char* what() const noexcept override { return "..."; } };

思路：noexcept 表示不抛异常，是必须的

RAII + 异常安全

代码：使用 unique_ptr / lock_guard 确保异常时资源正确释放

思路：智能指针和锁的析构函数会自动释放资源

noexcept 关键字

代码：void func() noexcept { ... }

思路：声明函数不抛异常，编译器可以优化
