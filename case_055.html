新语法精讲

Boost.Beast 的 I/O 模型

语法规则：Beast 基于 ASIO 的 Proactor 模式。io_context 是所有 I/O 的调度中心，socket/stream 都绑定在它上面。同步操作直接调用，异步操作通过回调/协程处理。
net::io_context ioc;  // I/O 调度器
tcp::socket sock(ioc);    // 创建绑定到 ioc 的 socket
// 同步：直接返回
net::connect(sock, ...);
// 异步：需要回调或协程
sock.async_connect(..., [](error_code ec) { ... });
ioc.run();  // 驱动异步事件循环

next_layer() 访问底层对象

语法规则：WebSocket stream 封装了 TCP socket。next_layer() 返回底层 socket 的引用，可以在 WebSocket 握手前/后进行 TCP 级别的操作（如设置超时、获取本地地址）。
ws::stream ws{ioc};
auto& tcp = ws.next_layer();          // 获取 TCP socket 引用
tcp.set_option(tcp::no_delay(true));  // 禁用 Nagle 算法
// 也可以用 beast::get_lowest_layer(ws)

beast::flat_buffer 动态缓冲区

语法规则：flat_buffer 是 ASIO 的动态缓冲区，读取数据时自动增长。适合未知大小的消息。make_printable 将缓冲区转为可打印形式。
beast::flat_buffer buf;
ws.read(buf);  // buf 自动调整大小
cout << beast::buffers_to_string(buf.data());  // 转为 string

举一反三

  • WebSocket 协议帧格式简析：
    WebSocket 帧格式：[FIN|RSV|opcode] [MASK|payload_len] [extended_len] [mask_key] [payload]
opcode: 0x1=文本, 0x2=二进制, 0x8=关闭, 0x9=Ping, 0xA=Pong
MASK: 客户端→服务端必须掩码，服务端→客户端不能掩码。

    理解帧格式有助于调试 WebSocket 问题，以及实现无依赖的轻量级客户端（如在嵌入式设备上）。

  • WebSocket 心跳保活 (Ping/Pong)：
    // 发送 Ping
ws.ping("");
// 读取 Pong（Beast 自动处理，但可手动控制）
ws.control_callback([](ws::frame_type kind, string_view payload) {
    if (kind == ws::frame_type::pong) cout << "Pong received";
});

    WebSocket 支持 Ping/Pong 帧用于心跳检测。Beast 默认自动处理，但可以设置自定义回调。
