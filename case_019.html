案例 19：shared_ptr/unique_ptr 实战

完整代码（带逐行注释）

1 #include <iostream>
2 #include <memory> 智能指针头文件
3 using namespace std;
4
5 class Resource {
6 int id_;
7 public:
8 Resource(int id) : id_(id) {
9 cout << "Resource " << id_ << " 创建" << endl;
10 }
11 ~Resource() {
12 cout << "Resource " << id_ << " 销毁" << endl;
13 }
14 void work() { cout << id_ << " 工作中" << endl; }
15 };
16
17 int main() {
18 cout << "=== unique_ptr（独占所有权）===" << endl;
19 { 块作用域
20 auto u1 = make_unique<Resource>(1); 创建 unique_ptr，自动管理 Resource 1
21 u1->work(); 用 -> 访问成员，和裸指针一样
22
23 auto u2 = move(u1); 所有权转移：u1 变为空，u2 持有 Resource 1
24 if (!u1) cout << "u1 已为空" << endl; unique_ptr 可以转为 bool 判断
25 u2->work(); u2 仍然可以工作
26 } 离开作用域，u2 自动销毁 Resource 1
27
28 cout << "\n=== shared_ptr（共享所有权）===" << endl;
29 {
30 auto s1 = make_shared<Resource>(2); 引用计数 = 1
31 cout << \"引用计数: \" << s1.use_count() << endl;
32 { 内层作用域
33 auto s2 = s1; 引用计数 = 2
34 cout << \"引用计数: \" << s1.use_count() << endl;
35 s2->work();
36 } s2 销毁，引用计数 = 1，Resource 2 不销毁
37 cout << \"引用计数: \" << s1.use_count() << endl;
38 } s1 销毁，引用计数 = 0，Resource 2 销毁
39
40 cout << "\n=== weak_ptr（不增加引用计数）===" << endl;
41 weak_ptr<Resource> wp; 空 weak_ptr
42 auto s3 = make_shared<Resource>(3);
43 wp = s3; weak_ptr 观察 s3，不增加计数
44 cout << \"use_count: \" << s3.use_count() << endl; 仍是 1
45
46 if (auto sp = wp.lock()) { lock() 返回 shared_ptr，若对象已销毁则返回空
47 sp->work(); 安全访问
48 }
49 s3.reset(); 手动释放 Resource 3
50 if (wp.expired()) cout << \"wp 指向的对象已销毁\" << endl; expired() 检查对象是否销毁
51
52 return 0;
53 }

新语法讲解

unique_ptr（独占所有权）

一个对象只能有一个 unique_ptr 持有它。不可拷贝，只能 move 转移。离开作用域自动 delete。优先使用 make_unique<T>(args...) 创建。
auto p = make_unique<int>(42);
auto p2 = move(p);  // p 变为 nullptr
// auto p3 = p;   // 编译错误，unique_ptr 不可拷贝

shared_ptr（共享所有权）与引用计数

多个 shared_ptr 可以指向同一对象，引用计数跟踪持有者数量。计数归零时自动 delete。用 use_count() 查看，make_shared<T>(args...) 创建。
auto s1 = make_shared<int>(100);  // count=1
auto s2 = s1;                      // count=2
s1.reset();                        // count=1
s2.reset();                        // count=0, 对象销毁

weak_ptr（打破循环引用）

weak_ptr 不增加引用计数，用于观察 shared_ptr 管理的对象。解决 shared_ptr 循环引用导致的内存泄漏问题。用 lock() 获取 shared_ptr。
// 循环引用场景：父对象持有子对象的 shared_ptr
// 子对象持有父对象的 shared_ptr → 永远回收不了
// 解决：子对象改用 weak_ptr 指向父对象

举一反三

自定义删除器

代码：auto p = unique_ptr(fopen(...), fclose);

思路：释放 FILE* 而不是 delete，适合 C API 资源

enable_shared_from_this

代码：class A : public enable_shared_from_this { auto ptr = shared_from_this(); };

思路：从对象内部安全获取自身的 shared_ptr

make_shared 的优势

代码：一次分配同时创建对象和控制块，比 new + shared_ptr 更高效且异常安全

思路：始终用 make_shared 代替 new
