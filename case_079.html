案例 79：内存泄漏检测器

重载全局 operator new/delete，在程序退出时报告所有未释放的内存，理解内存检测工具的底层原理。

核心知识点：operator new/delete 重载、noexcept、内存泄漏检测、mutex 线程安全、malloc/free

完整代码（含逐行注释）

// 案例 79：内存泄漏检测器
// 通过重载 new/delete 操作符实现轻量级内存泄漏检测。

#include <iostream>
#include <cstdlib>
#include <cstring>
#include <map>
#include <mutex>

/* 语法规则：重载全局 new/delete
   void* operator new(size_t size) — 分配内存
   void operator delete(void* ptr) noexcept — 释放内存
   void operator delete(void* ptr, size_t size) noexcept — C++14 带大小版本

   重载全局 new/delete 时要小心：
   1. 不能调用 malloc/free 之外的任何可能分配内存的函数（防止无限递归）
   2. 使用 std::mutex 保护全局状态
   3. 不要用 std::cout（它自己就用了 new）

   示例：跟踪分配/释放
     static std::map<void*, size_t> allocs;
     void* operator new(size_t sz) {
       void* p = malloc(sz);
       allocs[p] = sz;
       return p;
     }
     void operator delete(void* p) noexcept {
       allocs.erase(p);
       free(p);
     }

   举一反三：Visual Studio 的 CRT 调试堆（_CrtSetDbgFlag）、
   Valgrind（Linux）、AddressSanitizer 都是商业化级别的内存检测工具。
   本案例展示原理，生产环境请用专业工具。
*/

// 用静态变量存储分配记录（用 printf 避免 cout 递归调用 new）
struct AllocInfo {
    size_t size;
    const char* file;
    int line;
};

static std::map<void*, AllocInfo> allocMap;  // 地址 → 分配信息
static std::mutex allocMutex;                 // 线程安全
static size_t totalAllocated = 0;
static size_t totalFreed = 0;

// 重载 operator new
void* operator new(size_t size) {
    void* ptr = malloc(size);
    if (!ptr) throw std::bad_alloc();

    std::lock_guard<std::mutex> lock(allocMutex);
    allocMap[ptr] = {size, nullptr, 0};
    totalAllocated += size;
    return ptr;
}

// 重载 operator delete
void operator delete(void* ptr) noexcept {
    if (!ptr) return;

    std::lock_guard<std::mutex> lock(allocMutex);
    auto it = allocMap.find(ptr);
    if (it != allocMap.end()) {
        totalFreed += it->second.size;
        allocMap.erase(it);
    }
    free(ptr);
}

// 带大小版本的 delete（C++14）
void operator delete(void* ptr, size_t size) noexcept {
    if (!ptr) return;
    std::lock_guard<std::mutex> lock(allocMutex);
    allocMap.erase(ptr);
    totalFreed += size;
    free(ptr);
}

// 重载 new[] 和 delete[]
void* operator new[](size_t size) {
    return operator new(size);
}
void operator delete[](void* ptr) noexcept {
    operator delete(ptr);
}

// 泄漏报告
void reportLeaks() {
    std::lock_guard<std::mutex> lock(allocMutex);
    if (allocMap.empty()) {
        printf("\n[内存检测] 没有内存泄漏！\n");
    } else {
        printf("\n[内存检测] 发现 %zu 处内存泄漏:\n", allocMap.size());
        for (auto& [ptr, info] : allocMap) {
            printf("  地址 %p: %zu 字节\n", ptr, info.size);
        }
    }
    printf("总分配: %zu 字节 | 总释放: %zu 字节 | 泄漏: %zu 字节\n",
           totalAllocated, totalFreed, totalAllocated - totalFreed);
}

// ============ 测试代码 ============
class Widget {
    int data[100];  // 占用一定内存
public:
    Widget() { /*printf("Widget 构造\n");*/ }
    ~Widget() { /*printf("Widget 析构\n");*/ }
};

int main() {
    printf("案例 79：内存泄漏检测器\n");
    printf("========================\n");

    // 正常分配并释放
    int* a = new int(42);
    delete a;

    // 故意泄漏（注释掉 delete）
    int* leaked = new int[100];

    // Widget 数组
    auto* w1 = new Widget();
    auto* w2 = new Widget();
    delete w1;
    // delete w2;  ← 故意不释放，模拟泄漏

    printf("\n程序即将退出，运行泄漏检测...\n");
    reportLeaks();

    return 0;
}

语法规则：operator new/delete 重载

// 全局 operator new 的三个参数版本
void* operator new(size_t count);              // 基本版
void* operator new(size_t count, std::align_val_t al);  // C++17 对齐
// 类内也可以重载 new/delete（仅影响该类的分配）

💡 举一反三：重载 new/delete 的正当用途：内存池、泄漏检测、对齐控制、调试日志。注意：重载后必须调用 malloc/free，不能再用 new 分配内存（无限递归）。

语法规则：noexcept 说明符

void func() noexcept;         // 承诺不抛异常
void func() noexcept(true);    // 等价
void func() noexcept(false);   // 可能抛异常
// delete 必须是 noexcept 的，这是语言规定

💡 举一反三：noexcept 让编译器做优化（如用 move 代替 copy）。析构函数默认 noexcept(true)。移动构造函数应尽可能 noexcept。
