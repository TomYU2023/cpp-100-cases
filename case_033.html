新语法讲解

string_view 核心特性

string_view（C++17）是字符串的只读视图，包含一个指针+长度，不拥有数据。创建时不拷贝，传递时极轻量（通常 16 字节）。支持字符串的大部分只读操作（find、substr、compare 等），但不支持修改和空字符结尾。

举一反三

// string_view 适用场景
void process(string_view sv); // 可接收 string / char* / string_view
string s = "hello";
process(s);    // 隐式转换
process("hi"); // 字符串字面量
// 不可修改
sv[0] = 'H'; // 编译错误

string_view 的生命周期陷阱

string_view 不拥有数据，所以它指向的原始数据销毁后，string_view 变成悬空视图（dangling view），访问导致未定义行为。基本规则：string_view 的生命周期必须短于它指向的数据。

举一反三

// 危险写法
string_view getView() {
    string s = "hello";
    return s; // s 被销毁，返回悬空视图
}
// 安全写法
string_view getView(const string& s) { return s; }

实现 split_view 分割器

auto split(string_view sv, char delim) {
  vector parts;
  size_t start=0, end;
  while((end=sv.find(delim,start))!=string_view::npos) {
    parts.push_back(sv.substr(start,end-start));
    start=end+1;
  }
  parts.push_back(sv.substr(start));
  return parts;
}

零拷贝分割，适合解析 CSV/配置文件
