案例 33：string_view 零拷贝

使用 string_view 实现零拷贝的字符串传递和切片，减少不必要的内存分配。

核心知识点：string_view、零拷贝、生命周期管理

完整代码（含逐行注释）

// 案例 33：string_view 零拷贝
// 不拷贝字符串即可引用和切片
#include <iostream>
#include <string>
#include <string_view>
#include <vector>
using namespace std;

// 使用 string_view 作为参数（不拷贝字符串）
void printInfo(string_view sv) {
    cout << "内容: \"" << sv << "\""
         << "  长度: " << sv.size() << endl;
}

// 按分隔符拆分字符串（返回 string_view，不拷贝）
vector<string_view> split(string_view sv, char delim) {
    vector<string_view> result;
    size_t start = 0;
    size_t pos;

    while ((pos = sv.find(delim, start)) != string_view::npos) {
        result.push_back(sv.substr(start, pos - start));
        start = pos + 1;
    }
    // 最后一段
    if (start < sv.size()) {
        result.push_back(sv.substr(start));
    }
    return result;
}

int main() {
    // 1. 从 C 字符串构造
    string_view sv1 = "Hello, World!";
    printInfo(sv1);

    // 2. 从 std::string 构造
    string s = "C++ Programming";
    string_view sv2 = s;
    printInfo(sv2);

    // 3. 切片（O(1)，不分配新内存）
    string_view sub = sv2.substr(0, 3);
    cout << "子串: " << sub << endl; // "C++"

    // 4. remove_prefix / remove_suffix
    string_view sv3 = "   trim me   ";
    sv3.remove_prefix(3);  // 去掉前 3 个空格
    sv3.remove_suffix(3);  // 去掉后 3 个空格
    cout << "修剪后: \"" << sv3 << "\"" << endl;

    // 5. 拆分字符串
    string_view csv = "apple,banana,orange,grape";
    auto parts = split(csv, ',');
    cout << "拆分结果: ";
    for (auto p : parts) cout << p << " ";
    cout << endl;

    return 0;
}

语法规则：string_view 构造与传递

#include <string_view>
string_view sv = "hello";        // C字符串
string_view sv = std::string_obj;  // std::string
void f(string_view sv);           // 参数零拷贝

举一反三：string_view 只持有指针和长度，拷贝代价极低。可接受 C 字符串和 std::string，是函数参数的最佳实践。

语法规则：substr() 零拷贝切片

string_view sv = "hello world";
string_view sub = sv.substr(0, 5); // "hello"，不分配新内存

举一反三：string_view::substr 返回新的 view，只修改指针和长度。注意原始字符串的生命周期必须长于 view。

语法规则：string_view 生命周期陷阱

string_view bad() {
    string s = "temp";
    return s; // 危险！s 销毁后 view 悬垂
}

举一反三：string_view 不拥有数据，必须确保底层字符串在 view 使用期间存活。常见陷阱：临时 string、返回局部 string 的 view。
