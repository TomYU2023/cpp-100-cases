新语法精讲

ICMP 协议与 Ping 原理

语法规则：Ping 使用 ICMP（Internet Control Message Protocol）：发送 Echo Request（类型 8），目标回复 Echo Reply（类型 0）。通过计算 RTT 评估网络延迟。ICMP 不是基于端口的，所以不需要 TCP/UDP 端口号。
ICMP 报文格式：
| 类型(1B) | 代码(1B) | 校验和(2B) | 标识符(2B) | 序列号(2B) | 数据(变长) |
Echo Request:   类型=8, 代码=0
Echo Reply:     类型=0, 代码=0

TTL（Time To Live）生存时间

语法规则：TTL 是 IP 包头的字段，每经过一个路由器减 1，减到 0 则丢弃。初始 TTL 不同操作系统不同：Windows=128，Linux=64。通过收到的 TTL 可以推断目标 OS 类型。Ping 返回的 TTL 是目标机器的 TTL 减去路径跳数。
收到的 TTL = 目标 OS 默认 TTL - 路径跳数
TTL=114 → 可能 Windows (128-14)
TTL= 50 → 可能 Linux   (64-14)
// traceroute 就是利用 TTL 原理

Windows ICMP API

语法规则：IcmpCreateFile 创建 ICMP 句柄（不创建文件），IcmpSendEcho 同步发送 Echo Request 并等待回复。IcmpCloseHandle 关闭句柄。这是 Windows 特有的高层 API，比 Linux 的原始套接字方便很多。
// Linux 使用原始套接字
int sock = socket(AF_INET, SOCK_RAW, IPPROTO_ICMP);
// 需要 root 权限！
// 手动构造 ICMP 包头
struct icmphdr { uint8_t type, code; uint16_t checksum; ... };

举一反三

  • 平台兼容：用 RAII 抽象 ICMP 句柄：
    class IcmpSocket {
    HANDLE h = IcmpCreateFile();
public:
    IcmpSocket() = default;
    ~IcmpSocket() { if (h != INVALID_HANDLE_VALUE) IcmpCloseHandle(h); }
    int ping(const string& host) { /* ... */ }
};

    将 Windows 的 ICMP 句柄封装成 RAII 对象，确保异常时也能正确关闭。

  • Ping 监控 + 丢包率统计：
    while (true) {
    int sent = 0, received = 0;
    auto now = chrono::system_clock::now();
    // ... ping 一批包 ...
    double loss = 100.0 * (sent - received) / sent;
    cout << "[" << now << "] Loss: " << loss << "%\n";
    sleep(60);
}

    可以扩展成持续监控工具，记录延迟和丢包率，适合网络质量长期监控。
