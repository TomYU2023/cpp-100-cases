新语法精讲

定宽整数类型（cstdint）

语法规则：uint8_t（1 字节无符号）、uint32_t（4 字节）、uint64_t（8 字节）等类型保证固定宽度，不依赖任何平台。在协议实现、哈希算法、网络通信等对宽度敏感的场景必须使用定宽类型，不能用普通的 int。
uint8_t  byte  = 0xFF;     // 必定 1 字节
uint16_t word  = 0xFFFF;   // 必定 2 字节
uint32_t dword = 0xFFFFFFFF; // 必定 4 字节
// 普通 int 在 16/32/64 位平台上宽度不同！

hex + setw + setfill 格式化

语法规则：std::hex 切换为十六进制输出模式，std::setw(n) 设置最小输出宽度，std::setfill('0') 不足宽度时填充的字符。重点是：setw 是一次性的，每次输出后自动重置，需要反复设置。
int n = 15;
cout << hex << setw(4) << setfill('0') << n;
// 输出: 000f
cout << " " << n;
// 输出: f（setw 已重置！）
// 正确做法：每次都设置 setw
cout << setw(4) << a << setw(4) << b;

位运算：循环左移（Rotate Left）

语法规则：循环左移：把高位溢出的比特移回低位。(x << n) | (x >> (32-n))。左移 n 位得到高位部分，右移 32-n 位得到溢出的低位部分，两者 OR 合并。
uint32_t x = 0x80000001;  // 1000...0001
uint32_t y = (x << 1) | (x >> 31);
// x << 1  = 0x00000002
// x >> 31 = 0x00000001
// y = 0x00000003  (正确循环左移)

    实际项目中不要自己实现 MD5，使用 OpenSSL 或 Crypto++ 等成熟库。MD5 已不安全，新项目应使用 SHA-256。

  • 文件 MD5 校验（读取大文件）：
    ifstream file("data.bin", ios::binary);
string content((istreambuf_iterator(file)), istreambuf_iterator());
cout << md5.digest(content);

    结合文件流读取整个文件内容再计算哈希，实现完整的文件完整性校验工具。注意：大文件应分块读取以节省内存。
