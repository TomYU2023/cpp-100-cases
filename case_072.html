案例 72：装饰器模式

实现数据处理器管道：原始→加密→压缩→计时，装饰器可以任意嵌套组合，展示组合优于继承的设计原则。

核心知识点：装饰器模式、组合优于继承、继承构造函数 using、explicit、chrono 高精度计时

完整代码（含逐行注释）

// 案例 72：装饰器模式（Decorator Pattern）
// 装饰器模式：动态地给对象添加额外职责，比继承更灵活。
// 典型场景：咖啡加糖/加奶、日志包装器、性能计时、数据流加密/压缩。

#include <iostream>
#include <string>
#include <memory>
#include <chrono>

/* ── 语法规则：装饰器模式结构 ──
   Component（组件接口）：定义核心操作
   ConcreteComponent（具体组件）：实现核心功能
   Decorator（装饰器基类）：持有 Component 引用，转发调用
   ConcreteDecorator（具体装饰器）：在转发前后添加额外行为

   UML 示意：
     Component
        ▲
   ┌────┴────────┐
   │             │
   ConcreteComp  Decorator (has-a Component)
                     ▲
               ┌─────┴──────┐
               │            │
           DecoratorA    DecoratorB

   示例（咖啡）：
     Coffee 接口           ← cost() / description()
     SimpleCoffee          ← 基础咖啡，10元
     MilkDecorator         ← 加牛奶，+3元
     SugarDecorator        ← 加糖，+1元
     auto c = make_unique<SugarDecorator>(
              make_unique<MilkDecorator>(
              make_unique<SimpleCoffee>()));  // 嵌套装饰
     c->cost();  // 14元

   举一反三：装饰器组合比继承灵活得多。想加奶加糖用继承需要
   3 个子类（MilkCoffee/SugarCoffee/MilkSugarCoffee），
   装饰器只要组合即可。Python 的 @ 装饰器、Java 的 InputStream 包装都是这个模式。
*/

// 步骤 1：组件接口 — 数据处理
class DataProcessor {
public:
    virtual ~DataProcessor() = default;
    virtual std::string process(const std::string& data) = 0;
    virtual std::string description() const = 0;
};

// 步骤 2：具体组件 — 原始数据处理（什么都不做）
class RawProcessor : public DataProcessor {
public:
    std::string process(const std::string& data) override {
        return data;  // 原样返回
    }
    std::string description() const override {
        return "原始数据";
    }
};

// 步骤 3：装饰器基类 — 持有被装饰对象
class ProcessorDecorator : public DataProcessor {
protected:
    std::unique_ptr<DataProcessor> wrappee;  // 被装饰的对象

public:
    explicit ProcessorDecorator(std::unique_ptr<DataProcessor> p)
        : wrappee(std::move(p)) {}

    // 默认转发
    std::string process(const std::string& data) override {
        return wrappee->process(data);
    }
    std::string description() const override {
        return wrappee->description();
    }
};

// 步骤 4：具体装饰器 1 — 加密（Base64 风格编码）
class EncryptDecorator : public ProcessorDecorator {
public:
    using ProcessorDecorator::ProcessorDecorator;  // 继承构造函数

    std::string process(const std::string& data) override {
        std::string raw = wrappee->process(data);  // 先让下层处理
        // 简单"加密"：每个字符加 1
        std::string encrypted;
        for (char c : raw) encrypted += static_cast<char>(c + 1);
        return encrypted;
    }

    std::string description() const override {
        return wrappee->description() + " → 加密";
    }
};

// 步骤 4：具体装饰器 2 — 压缩（模拟：移除空格）
class CompressDecorator : public ProcessorDecorator {
public:
    using ProcessorDecorator::ProcessorDecorator;

    std::string process(const std::string& data) override {
        std::string raw = wrappee->process(data);
        std::string compressed;
        for (char c : raw) {
            if (c != ' ') compressed += c;  // 移除空格
        }
        return compressed;
    }

    std::string description() const override {
        return wrappee->description() + " → 压缩";
    }
};

// 步骤 4：具体装饰器 3 — 性能计时（不修改数据，只记录时间）
class TimerDecorator : public ProcessorDecorator {
public:
    using ProcessorDecorator::ProcessorDecorator;

    std::string process(const std::string& data) override {
        auto start = std::chrono::high_resolution_clock::now();  // 记录开始时间

        std::string result = wrappee->process(data);             // 执行实际处理

        auto end = std::chrono::high_resolution_clock::now();    // 记录结束时间
        auto duration = std::chrono::duration_cast<
            std::chrono::microseconds>(end - start).count();      // 计算耗时(微秒)

        std::cout << "[计时] 处理耗时: " << duration << " μs" << std::endl;
        return result;
    }

    std::string description() const override {
        return wrappee->description() + " → 计时";
    }
};

int main() {
    std::string input = "Hello World from C++";

    // 场景 1：原始处理
    auto raw = std::make_unique<RawProcessor>();
    std::cout << "[" << raw->description() << "]" << std::endl;
    std::cout << "输入 : " << input << std::endl;
    std::cout << "输出 : " << raw->process(input) << "\n\n";

    // 场景 2：加密 + 压缩（装饰器嵌套）
    // 处理顺序：压缩 → 加密（从内到外）
    auto pipeline = std::make_unique<EncryptDecorator>(
                        std::make_unique<CompressDecorator>(
                            std::make_unique<RawProcessor>()));

    std::cout << "[" << pipeline->description() << "]" << std::endl;
    std::cout << "输入 : " << input << std::endl;
    std::cout << "输出 : " << pipeline->process(input) << "\n\n";

    // 场景 3：计时 + 原始处理（监控性能）
    auto monitored = std::make_unique<TimerDecorator>(
                         std::make_unique<RawProcessor>());

    std::cout << "[" << monitored->description() << "]" << std::endl;
    monitored->process("Benchmark test string");

    return 0;
}

语法规则：继承构造函数 using

class Base {
public:
  Base(int x, int y);
};
class Derived : public Base {
public:
  using Base::Base;  // 继承基类所有构造函数,不用手写转发
};

💡 举一反三：C++11 引入 using 继承构造函数，省去写大量转发代码。注意：继承构造函数不会自动初始化派生类自己的成员

语法规则：explicit 防止隐式转换

class MyClass {
public:
  explicit MyClass(int x);  // 禁止 MyClass obj = 5; 这种隐式转换
};
// 必须写成 MyClass obj(5); 或 MyClass obj{5};

💡 举一反三：explicit 写在单参数构造函数前，防止意外的隐式类型转换。C++ 核心指南建议：所有单参数构造函数都应加 explicit。

语法规则：chrono 高精度计时

auto start = std::chrono::high_resolution_clock::now();
// ... 被测代码 ...
auto end = std::chrono::high_resolution_clock::now();
auto us = duration_cast<microseconds>(end - start).count();

💡 举一反三：chrono 是 C++11 引入的类型安全时间库。duration_cast 做精度转换，count() 返回数值。steady_clock 适合测量时间间隔（不受系统时间调整影响）。
