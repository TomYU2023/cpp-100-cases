新语法精讲

Windows 句柄（HANDLE）与 RAII 封装

语法规则：HANDLE 是 Windows 中内核对象的抽象标识（void*）。文件、目录、事件、线程、进程都用 HANDLE 表示。释放用 CloseHandle。C++ 中应该用 unique_ptr 配合自定义删除器管理。
// RAII HANDLE 封装
struct HandleDeleter {
    using pointer = HANDLE;
    void operator()(HANDLE h) const {
        if (h != INVALID_HANDLE_VALUE && h != nullptr)
            CloseHandle(h);
    }
};
using UniqueHandle = unique_ptr;

FILE_FLAG_BACKUP_SEMANTICS 的意义

语法规则：CreateFile 默认只打开文件。要打开目录，必须加 FILE_FLAG_BACKUP_SEMANTICS。这个标志名容易误导，含义是"允许不备份权限的用户打开对象"——它的副作用是允许打开目录。
// 打开文件
HANDLE hFile = CreateFile("data.txt", ..., OPEN_EXISTING, 0, nullptr);
// 打开目录（加 FILE_FLAG_BACKUP_SEMANTICS）
HANDLE hDir  = CreateFile("C:\\dir", ..., OPEN_EXISTING,
                          FILE_FLAG_BACKUP_SEMANTICS, nullptr);

FILE_NOTIFY_INFORMATION 链表遍历

语法规则：ReadDirectoryChangesW 返回 FILE_NOTIFY_INFORMATION 链表。每个节点的 NextEntryOffset 指向下一个（0 表示结束）。文件名是 WCHAR 数组，长度以字符计（非字节），需用 WideCharToMultiByte 转换。
FILE_NOTIFY_INFORMATION* node = (FILE_NOTIFY_INFORMATION*)buffer;
while (true) {
    wstring wname(node->FileName, node->FileNameLength / sizeof(WCHAR));
    // 处理 node->Action (FILE_ACTION_ADDED 等)
    if (node->NextEntryOffset == 0) break;
    node = (FILE_NOTIFY_INFORMATION*)((BYTE*)node + node->NextEntryOffset);
}
