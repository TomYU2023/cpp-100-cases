案例 27：sort/stable_sort对比

对比 sort 和 stable_sort 在处理相等元素时的行为差异，深入理解排序算法稳定性。

核心知识点：sort、stable_sort、稳定性、自定义比较器

完整代码（含逐行注释）

// 案例 27：sort / stable_sort 对比
// 演示稳定排序与不稳定排序的区别
#include <iostream>
#include <vector>
#include <algorithm>
#include <string>
using namespace std;

struct Person {
    string name;
    int grade;
};

// 按 grade 排序的比较器
bool byGrade(const Person& a, const Person& b) {
    return a.grade < b.grade;
}

void printList(const string& label, const vector<Person>& people) {
    cout << label << ":" << endl;
    for (const auto& p : people) {
        cout << "  " << p.name << " : " << p.grade << "分" << endl;
    }
    cout << endl;
}

int main() {
    // 原始数据：Alice 和 Charlie 成绩相同
    vector<Person> people = {
        {"Alice",   85},
        {"Bob",     92},
        {"Charlie", 85},
        {"Dave",    78},
        {"Eve",     92}
    };

    printList("原始顺序", people);

    // 非稳定排序：相同 grade 时，原始顺序不保证
    vector<Person> sorted = people;
    sort(sorted.begin(), sorted.end(), byGrade);
    printList("sort (非稳定)", sorted);

    // 稳定排序：相同 grade 时，保持原始相对顺序
    vector<Person> stableSorted = people;
    stable_sort(stableSorted.begin(), stableSorted.end(), byGrade);
    printList("stable_sort (稳定)", stableSorted);

    return 0;
}

自定义比较器

【规则】sort/priority_queue/set 等 STL 容器和算法接受自定义比较器——函数指针、函数对象（functor）或 lambda 表达式。比较器必须满足严格弱序（strict weak ordering）。

【原理】比较器 comp(a, b) 返回 true 表示 a 应排在 b 前面。STL 的所有排序/查找算法都基于 < 操作构建，不依赖 ==。这是为了通用性——某些类型只有偏序关系（如任务优先级中可能有两个任务同等优先），严格全序反而难以建模。比较器需满足：irreflexive、transitive、比较等价传递性。

【常见陷阱】1. 将小于等于比较逻辑（<=）引入比较器，使得 comp(a,a) 返回 true——违反 irreflexive 原则，sort 会越界访问；2. 使用未初始化的变量作为排序依据——导致同一元素多次比较结果不同，sort 进入未定义行为。

bool cmp(const T& a, const T& b) {
    return a.field < b.field;
}
sort(v.begin(), v.end(), cmp);
