新语法讲解

sort vs stable_sort

sort：Introsort（快排+堆排+插入排序混合），不保证稳定性，O(n log n)。stable_sort：归并排序或类似算法，保持相等元素的相对顺序，O(n log n) 但常数更大，需要额外 O(n) 内存。

举一反三

// 稳定排序示例
vector> v = {{3,"c"},{1,"a"},{3,"b"}};
sort(v.begin(), v.end()); // 可能: (1,"a"),(3,"b"),(3,"c") 或 (3,"c"),(3,"b")
stable_sort(v.begin(),v.end()); // 保证: (1,"a"),(3,"c"),(3,"b")

partial_sort 部分排序

partial_sort(first, middle, last)：只把最小的 middle-first 个元素排好放到前面，其余元素顺序未定义。适用于只需要 Top K 的场景，比全排序快。

举一反三

vector v = {9,3,7,1,5,8};
partial_sort(v.begin(), v.begin()+3, v.end());
// 前 3 个: {1,3,5}（最小 3 个且已排序）
// 后面: {9,7,8}（顺序未定义）

举一反三

nth_element 找第 K 大

nth_element(v.begin(), v.begin()+k, v.end(), greater<>());
int kth = v[k]; // 第 k 大的值
// 左边都 >= kth，右边都 <= kth，但不排序
