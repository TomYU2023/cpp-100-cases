案例 59：原子操作与内存序

使用 atomic 实现无锁计数器，理解内存序的基本概念。

核心知识点：atomic、memory_order、CAS、无锁编程

完整代码（含逐行注释）

// 案例 59：原子操作与内存序
// lock-free 计数器和标志
#include <iostream>
#include <atomic>
#include <thread>
#include <vector>
#include <chrono>
using namespace std;

// 无锁计数器
class AtomicCounter {
private:
    atomic<long long> count{0};

public:
    // fetch_add 原子地增加并返回旧值
    long long increment() {
        return count.fetch_add(1, memory_order_relaxed);
    }

    long long get() const { return count.load(memory_order_relaxed); }
};

// 自旋锁（使用 atomic_flag）
class SpinLock {
private:
    atomic_flag flag = ATOMIC_FLAG_INIT;

public:
    void lock() {
        // 自旋等待，直到成功设置标志
        while (flag.test_and_set(memory_order_acquire)) {
            // 让出 CPU（可选，减少忙等开销）
            // this_thread::yield();
        }
    }

    void unlock() {
        flag.clear(memory_order_release);
    }
};

// 无锁的「只执行一次」标志
class RunOnce {
private:
    atomic<bool> executed{false};

public:
    // 尝试执行，如果已经执行过则返回 false
    bool tryRun(function<void()> func) {
        bool expected = false;
        // CAS：如果 expected 和当前值相等，则设置为 true
        if (executed.compare_exchange_strong(expected, true)) {
            func(); // 只有第一次调用会执行
            return true;
        }
        return false;
    }
};

int main() {
    // 1. 原子计数器
    const int N = 8;
    const int M = 100000;
    AtomicCounter counter;

    vector<thread> threads;
    for (int i = 0; i < N; i++) {
        threads.emplace_back([&counter, M]() {
            for (int j = 0; j < M; j++) {
                counter.increment(); // 无锁递增
            }
        });
    }
    for (auto& t : threads) t.join();

    cout << "计数器: " << counter.get()
         << " (期望: " << N * M << ")" << endl;

    // 2. 只执行一次
    RunOnce once;
    auto initFunc = []() { cout << "初始化（只执行一次）" << endl; };
    cout << boolalpha;
    cout << "tryRun 1: " << once.tryRun(initFunc) << endl;
    cout << "tryRun 2: " << once.tryRun(initFunc) << endl;
    cout << "tryRun 3: " << once.tryRun(initFunc) << endl;

    return 0;
}

语法规则：atomic 原子操作

#include <atomic>
atomic<int> count{0};
count.fetch_add(1);   // 原子递增
count.load();          // 原子读取
count.store(42);       // 原子写入

举一反三：atomic 保证操作不可分割。对 int 的 ++ 是三个步骤（读-改-写），非原子操作会因线程交错而出错。常用操作：fetch_add/sub/and/or/xor。

语法规则：compare_exchange (CAS)

bool expected = false;
if (flag.compare_exchange_strong(expected, true)) {
    // CAS 成功，执行关键操作
}
// 失败时 expected 被更新为当前值

举一反三：CAS (Compare And Swap) 是无锁编程的基础原语。compare_exchange_strong 用循环重试，compare_exchange_weak 可能假失败需配合循环。适合实现无锁栈/队列。

语法规则：memory_order 内存序

memory_order_relaxed: 只保证原子性，不保证顺序
memory_order_acquire: 后续读写不重排到此操作前
memory_order_release: 之前的读写不重排到此操作后
memory_order_seq_cst: 全局顺序一致性（默认，最安全最慢）

举一反三：默认 seq_cst 最易理解但性能最低。计数器用 relaxed 即可。acquire-release 配对用于生产者-消费者同步。错误的内存序会导致难以调试的并发 bug。
