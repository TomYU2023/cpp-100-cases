案例 77：静态库与动态库打包发布

展示跨平台符号导出宏、CMake install 规则、以及如何生成 find_package 配置文件。

核心知识点：静态库/动态库、dllexport/dllimport、CMake install、find_package 集成

完整代码（含逐行注释）

// 案例 77：静态库与动态库打包发布
// 展示如何将 C++ 代码编译为库（.lib / .a / .dll / .so）并供其他项目使用。

#include <iostream>

/* 语法规则：静态库 vs 动态库
   静态库 (.lib / .a)：
     - 编译时链接到可执行文件中
     - 运行时不依赖外部文件
     - 缺点：每个程序都有一份副本，总体积大，更新需重新编译

   动态库 (.dll / .so)：
     - 运行时加载，多个程序共享一份
     - 优点：节省磁盘和内存，可热更新
     - 缺点：部署复杂（DLL Hell），需要版本管理

   符号导出（Windows）：
     __declspec(dllexport) — 导出符号
     __declspec(dllimport) — 导入符号
     用宏统一：
       #ifdef BUILD_MYLIB
       #define MYLIB_API __declspec(dllexport)
       #else
       #define MYLIB_API __declspec(dllimport)
       #endif

   示例：CMake 构建库
     add_library(mylib STATIC src/foo.cpp)   # 静态库
     add_library(mylib SHARED src/foo.cpp)   # 动态库
     target_compile_definitions(mylib PRIVATE BUILD_MYLIB)  # 定义宏

   举一反三：大型项目通常提供静态库用于测试和内部工具，
   动态库用于对外发布的 SDK 和插件系统。
*/

// ============ 库头文件：mylib/mathlib.h ============
/*
// 跨平台符号导出宏
#ifdef _WIN32
    #ifdef MATHLIB_BUILD
        #define MATHLIB_API __declspec(dllexport)  // 构建时导出
    #else
        #define MATHLIB_API __declspec(dllimport)  // 使用时导入
    #endif
#else
    #define MATHLIB_API  // Linux/macOS 默认所有符号可见
#endif

namespace mathlib {
    // 使用 API 宏标记公开接口
    MATHLIB_API int factorial(int n);
    MATHLIB_API double power(double base, int exp);
}
*/

// ============ 库源文件：mylib/mathlib.cpp ============
/*
#include "mathlib.h"

namespace mathlib {
    int factorial(int n) {
        if (n <= 1) return 1;
        return n * factorial(n - 1);
    }

    double power(double base, int exp) {
        if (exp == 0) return 1.0;
        double result = 1.0;
        for (int i = 0; i < exp; ++i)
            result *= base;
        return result;
    }
}
*/

// ============ CMakeLists.txt（库项目）============
/*
cmake_minimum_required(VERSION 3.16)
project(MathLib VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)

# 静态库版本
add_library(mathlib_static STATIC src/mathlib.cpp)
target_include_directories(mathlib_static PUBLIC include)

# 动态库版本
add_library(mathlib_shared SHARED src/mathlib.cpp)
target_include_directories(mathlib_shared PUBLIC include)
target_compile_definitions(mathlib_shared PRIVATE MATHLIB_BUILD)

# 安装规则
install(TARGETS mathlib_static mathlib_shared
    LIBRARY DESTINATION lib    # .so
    ARCHIVE DESTINATION lib    # .a / .lib
    RUNTIME DESTINATION bin    # .dll
)
install(DIRECTORY include/ DESTINATION include)  # 头文件

# 生成 CMake 配置文件，方便 find_package
include(CMakePackageConfigHelpers)
configure_package_config_file(
    MathLibConfig.cmake.in
    ${CMAKE_CURRENT_BINARY_DIR}/MathLibConfig.cmake
    INSTALL_DESTINATION lib/cmake/MathLib
)
install(FILES ${CMAKE_CURRENT_BINARY_DIR}/MathLibConfig.cmake
    DESTINATION lib/cmake/MathLib)
*/

int main() {
    std::cout << "案例 77：静态库与动态库打包发布" << std::endl;
    std::cout << "================================" << std::endl;
    std::cout << "关键概念：" << std::endl;
    std::cout << "  __declspec(dllexport) — 导出符号到 DLL" << std::endl;
    std::cout << "  __declspec(dllimport) — 从 DLL 导入符号" << std::endl;
    std::cout << "  install() — CMake 安装规则" << std::endl;
    std::cout << "  configure_package_config_file — find_package 支持" << std::endl;
    std::cout << "\n库消费者使用方式：" << std::endl;
    std::cout << "  find_package(MathLib REQUIRED)" << std::endl;
    std::cout << "  target_link_libraries(myapp PRIVATE MathLib::mathlib_static)" << std::endl;
    return 0;
}

语法规则：__declspec(dllexport/dllimport) Windows 符号导出

// Windows DLL 符号导出宏惯用模式
#ifdef _WIN32
  #ifdef MYLIB_EXPORTS
    #define MYLIB_API __declspec(dllexport)
  #else
    #define MYLIB_API __declspec(dllimport)
  #endif
#else
  #define MYLIB_API  // Linux 不需要
#endif

💡 举一反三：Windows 的 DLL 默认不导出符号，必须显式标记。Linux 默认全部导出（可用 -fvisibility=hidden 限制）。CMake 的 GENERATE_EXPORT_HEADER 可自动生成导出宏。
