案例 65：TCP 聊天室（服务端 + 客户端）

阶段：工程能力锻造难度：⭐⭐⭐⭐练什么：socket、select、协议设计
实现一个多人 TCP 聊天室：服务端用 select 管理多客户端连接，广播消息给所有人。理解 TCP 网络编程全流程。

📖 语法规则：BSD Socket API 流程
socket() → bind() → listen() → accept() → send()/recv() → close()

示例：

int fd = socket(AF_INET, SOCK_STREAM, 0);
bind(fd, (sockaddr*)&addr, sizeof(addr));
listen(fd, 5);
int client = accept(fd, nullptr, nullptr);
send(client, msg, len, 0);
recv(client, buf, sizeof(buf), 0);
closesocket(client); closesocket(fd);

💡 举一反三：

  • Windows 需要 WSAStartup/WSACleanup，Linux 不需要
  • send/recv 返回值可能是部分发送/接收，要循环处理
  • TCP 是流式协议，消息边界需要在应用层定义（如长度前缀）
📖 语法规则：select 多路复用
select(maxfd+1, &readfds, NULL, NULL, &timeout);

示例：

fd_set readfds;
FD_ZERO(&readfds);
FD_SET(fd, &readfds);
int n = select(maxfd+1, &readfds, NULL, NULL, NULL);
if (FD_ISSET(fd, &readfds)) { /* readable */ }

💡 举一反三：

  • select 有 fd 数量上限（默认 1024），Windows 上无此限制但对 socket 有效
  • Linux 上 epoll / Windows 上 IOCP 是 select 的升级替代
  • select 每次调用都要重新设置 fd_set（因为会被修改）
