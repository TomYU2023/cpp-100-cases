案例 65：std::variant 类型安全联合

使用 variant 替代 union，实现类型安全的联合体。

核心知识点：variant、visit、get、holds_alternative

完整代码（含逐行注释）

// 案例 65：std::variant 类型安全联合
// 安全的多类型联合体
#include <iostream>
#include <variant>
#include <string>
#include <vector>
#include <cmath>
using namespace std;

// 可表示整数、浮点数或字符串的值
using Value = variant<int, double, string>;

// 使用 visit 处理 variant
void printValue(const Value& v) {
    visit([](const auto& val) {
        using T = decay_t<decltype(val)>;
        if constexpr (is_same_v<T, int>) {
            cout << "整数: " << val << endl;
        } else if constexpr (is_same_v<T, double>) {
            cout << "浮点数: " << fixed << setprecision(2) << val << endl;
        } else if constexpr (is_same_v<T, string>) {
            cout << "字符串: \"" << val << "\"" << endl;
        }
    }, v);
}

// 对 variant 执行运算
Value compute(Value a, Value b, char op) {
    return visit([op](auto&& x, auto&& y) -> Value {
        using T = common_type_t<decay_t<decltype(x)>,
                                 decay_t<decltype(y)>>;
        if constexpr (is_arithmetic_v<T>) {
            switch (op) {
                case '+': return T(x + y);
                case '-': return T(x - y);
                case '*': return T(x * y);
                case '/': return (y != 0) ? Value(T(x / y))
                                          : Value(string("除零错误"));
            }
        }
        return Value(string("不支持的操作"));
    }, a, b);
}

int main() {
    // 创建不同类型的值
    Value v1 = 42;
    Value v2 = 3.14;
    Value v3 = string("Hello");

    printValue(v1);
    printValue(v2);
    printValue(v3);

    // 检查当前存储的类型
    cout << "\nv1 是 int? " << holds_alternative<int>(v1) << endl;
    cout << "v1 是 double? " << holds_alternative<double>(v1) << endl;

    // 获取值
    cout << "get<int>(v1) = " << get<int>(v1) << endl;

    // 计算
    auto result = compute(10, 3.5, '+');
    printValue(result);

    result = compute(string("hi"), string("!"), '+');
    printValue(result);

    return 0;
}

语法规则：variant 基本操作

variant<int,double,string> v;
v = 42;                // 存 int
v = 3.14;              // 改为 double
get<int>(v);           // 获取（类型不对抛 bad_variant_access）
holds_alternative<int>(v); // 判断当前类型

举一反三：variant 是类型安全的 union，自动管理类型切换时析构/构造。编译期知道所有可能类型。比 any 更快（无堆分配）但类型集合固定。

语法规则：visit 模式匹配

visit([](const auto& val) {
    if constexpr (is_same_v<T, int>) // 处理 int
    else if constexpr (is_same_v<T, double>) // 处理 double
}, v);

举一反三：visit 是安全访问 variant 的首选方式。编译器保证所有类型都被处理（否则编译错误）。if constexpr 在编译期选择分支，避免了运行时开销。overload 模式可让代码更简洁。

语法规则：variant vs union vs any

// union: 不安全（需手动跟踪类型）
// variant: 类型安全 + 编译期类型集 + 栈分配
// any: 任意类型 + 运行时 + 堆分配

举一反三：variant 适合「固定几种类型切换」场景（如 JSON 值、AST 节点）。any 适合「未知类型」场景（如配置容器）。优先用 variant 追求编译期安全。
