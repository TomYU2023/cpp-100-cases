案例 12：银行账户类(封装+友元)

设计一个银行账户类，练习访问控制（private/public）、封装、友元函数和静态成员。

核心知识点：访问控制、友元、静态成员、const 成员

完整代码（含逐行注释）

// 案例 12：银行账户类（封装 + 友元）
// 练习 private/public、友元函数、静态成员
#include <iostream>
#include <string>
using namespace std;

class BankAccount {
private:
    string owner;         // 账户持有人
    string accountId;     // 账号
    double balance;       // 余额

    // 静态成员：银行总存款
    static double totalDeposits;

public:
    BankAccount(const string& name, const string& id, double initial)
        : owner(name), accountId(id), balance(initial) {
        totalDeposits += initial;  // 存入计入总存款
    }

    ~BankAccount() {
        totalDeposits -= balance;  // 销毁时从总存款扣除
    }

    // 存款（公有接口）
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            totalDeposits += amount;
            cout << "存款 " << amount << " 元成功" << endl;
        }
    }

    // 取款（公有接口，有余额检查）
    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            totalDeposits -= amount;
            cout << "取款 " << amount << " 元成功" << endl;
            return true;
        }
        cout << "余额不足！" << endl;
        return false;
    }

    // const 成员函数：获取余额（承诺不修改成员）
    double getBalance() const { return balance; }
    string getOwner() const { return owner; }

    // 友元函数：允许访问私有成员，用于转账
    friend bool transfer(BankAccount& from, BankAccount& to, double amount) {
        if (amount > 0 && from.balance >= amount) {
            from.balance -= amount;
            to.balance += amount;
            cout << "转账 " << amount << " 元："
                 << from.owner << " -> " << to.owner << endl;
            return true;
        }
        cout << "转账失败：余额不足" << endl;
        return false;
    }

    // 静态成员函数
    static double getTotalDeposits() { return totalDeposits; }
};

// 静态成员必须在类外定义
double BankAccount::totalDeposits = 0;

int main() {
    BankAccount alice("Alice", "A001", 5000);
    BankAccount bob("Bob", "B001", 3000);

    // 存取款操作
    alice.deposit(2000);
    alice.withdraw(1000);

    // 转账（通过友元函数）
    transfer(alice, bob, 1500);

    cout << "Alice 余额: " << alice.getBalance() << endl;
    cout << "Bob 余额: " << bob.getBalance() << endl;
    cout << "银行总存款: " << BankAccount::getTotalDeposits() << endl;

    return 0;
}

运行结果

存款 2000 元成功
取款 1000 元成功
转账 1500 元：Alice -> Bob
Alice 余额: 4500
Bob 余额: 4500
银行总存款: 9000

访问控制: private vs public

【规则】public 成员可被外部代码访问，private 成员只能在类内部（包括友元）访问。这是 C++ 封装的核心机制，隐藏实现细节。适用于所有需要数据保护的类。

【原理】访问控制是编译期概念——编译器在编译时检查访问权限，不产生任何运行时开销。private 不是「安全」机制（反射、friend、#define private public 等方式可以绕过），而是「意图表达」机制——告诉使用者「这是内部实现细节，变更后不保证兼容」。封装的价值在于可以自由修改内部实现而不影响外部代码。

【常见陷阱】1. 将所有成员变量设为 public 以方便外部访问——破坏了封装性，后期修改数据结构时所有使用方都要改；2. 为每个 private 变量都写了 getter/setter 但对 setter 不做任何验证——这本质上和 public 没有区别，只是多了函数调用开销。

class MyClass {
private:
    int secret; // 只能类内访问
public:
    int getSecret() const { return secret; } // 公共接口
};

友元函数

【规则】friend 关键字授予非成员函数或另一个类访问本类 private 成员的权限。适用于操作符重载、紧密协作的类之间、测试代码等场景。

【原理】友元破坏了封装——这是设计上的刻意妥协。C++ 的设计哲学是「你可以打破规则，但必须显式声明」。友元关系是单向的、不可传递的、不可继承的。适度的友元比把所有成员设为 public 更好——至少它精确声明了谁可以访问隐密数据。最佳实践：将友元数限制在最小范围。

【常见陷阱】1. 滥用 friend 让一个类可以访问另一个类的全部私有成员——友元应该是最小权限的「外科手术式」访问；2. 以为 friend class B 使得 B 的派生类也能访问私有成员——友元关系不继承，派生类没有权限。

class A {
    friend void func(A& a); // func 可访问 A 的私有成员
};

静态成员变量与函数

【规则】static 成员变量属于类本身而非对象（所有对象共享同一份数据），static 成员函数可不通过对象调用。适用于计数器、配置常量、工厂方法等类级别操作。

【原理】静态成员变量在类定义中只是声明——必须在类外（.cpp 文件中）定义一次以分配存储空间。C++17 引入 inline static 成员变量，可直接在类定义中初始化，省去类外定义。静态成员函数没有 this 指针，因此不能访问非静态成员，也不能是 virtual。

【常见陷阱】1. 在头文件的类定义中初始化了非 const 静态成员变量——编译通过但链接报多重定义错误；2. 在多线程环境中，多个线程同时修改同一静态成员变量而没有同步机制——数据竞争导致未定义行为。

class C {
    static int count;  // 类级别共享
public:
    static int getCount() { return count; }
};
int C::count = 0; // 类外定义

【变体练习一】为案例 012 的 BankAccount 类添加静态成员 totalAccounts 和 nextAccountNumber，每次构造新账户时自动递增编号。添加静态函数 getTotalAccounts() 查询总数。

【变体练习二】使用 C++17 的 inline static 成员变量改写 totalAccounts 的定义，省去类外定义。然后用 thread_local static 变量实现每线程独立的计数器。
