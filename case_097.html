案例 97：std::apply tuple 调用

使用 apply 将 tuple 解包为函数参数，实现运行时动态参数调用。

核心知识点：apply、tuple 解包、变参转发、通用调用

完整代码（含逐行注释）

// 案例 97：std::apply tuple 调用
// 将 tuple 元素展开为函数参数
#include <iostream>
#include <tuple>
#include <functional>
#include <string>
#include <vector>
using namespace std;

// 接受 3 个参数的函数
void printPerson(const string& name, int age, double height) {
    cout << "姓名: " << name
         << ", 年龄: " << age
         << ", 身高: " << height << "m" << endl;
}

int sum(int a, int b, int c, int d) {
    return a + b + c + d;
}

int main() {
    // 1. 基本用法：tuple 展开为函数参数
    auto person = make_tuple("Alice", 25, 1.68);
    apply(printPerson, person);

    auto nums = make_tuple(1, 2, 3, 4);
    cout << "sum = " << apply(sum, nums) << endl;

    // 2. apply 配合 Lambda
    auto data = make_tuple(10, 20, 30);

    // 展开 tuple 到 Lambda 参数
    auto result = apply([](int x, int y, int z) {
        return x * y + z;
    }, data);
    cout << "10*20+30 = " << result << endl;

    // 3. 动态调用：根据条件构建参数
    vector<tuple<string, int, double>> people = {
        {"Bob", 30, 1.75},
        {"Charlie", 22, 1.82},
        {"David", 28, 1.70}
    };

    cout << "\n=== 批量调用 ===" << endl;
    for (const auto& p : people) {
        apply(printPerson, p);
    }

    // 4. apply + invoke 的组合（元编程常见）
    // 从 tuple 中调用对象的成员函数
    struct Task {
        void execute(int id, const string& msg) {
            cout << "Task " << id << ": " << msg << endl;
        }
    };

    Task task;
    auto params = make_tuple(42, "处理完成");
    apply([&task](auto&&... args) {
        task.execute(forward<decltype(args)>(args)...);
    }, params);

    return 0;
}

语法规则：std::apply

#include <tuple>
apply(func, myTuple);
// 等价于 func(get<0>(t), get<1>(t), ..., get<N-1>(t))
// func 和 tuple 的元素数量必须匹配

举一反三：apply (C++17) 是 tuple 到函数调用的桥梁。类似 Python 的 func(*args)。参数数量和类型在编译期检查。常用于延迟调用和参数存储。

语法规则：apply + Lambda

apply([](int x, int y) { return x + y; }, make_tuple(1, 2));
// Lambda 的参数从 tuple 展开
// 不需要显式指定 Lambda 参数类型（从 tuple 推导）

举一反三：apply+Lambda 是最常见的组合。比直接传 tuple 到算法更灵活（因为算法通常需要原始参数）。可以配合完美转发（forward）实现高效参数传递。

语法规则：apply 与 invoke 的区别

// apply: 将 tuple 展开为函数的多个参数
apply(func, tuple);
// invoke: 统一调用（函数/成员函数/函数对象）
invoke(func, arg1, arg2);
// 两者组合: apply(invoke, tuple(成员函数, 对象, 参数...))

举一反三：apply 解决「参数打包」问题。invoke 解决「调用方式统一」问题。两者正交互补。make_from_tuple 类似 apply 但用于构造对象。
