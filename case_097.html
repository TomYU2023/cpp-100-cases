案例 97：LLVM Pass 插件

编写一个 LLVM Function Pass，在编译期统计每个函数的 IR 指令数量，理解编译器插件化和 IR 遍历机制。

核心知识点：LLVM Pass 架构、FunctionPass 继承、IR 遍历（BB->Inst）、RegisterPass 注册

完整代码（含逐行注释）

// 案例 97：LLVM Pass 插件 —— 编译器插件的入门
// LLVM Pass 可以插入到编译流水线中，分析和转换 IR（中间表示）。
// 本案例实现 Function Pass：统计每个函数的指令数量。
//
// 编译: clang++ -shared -fPIC FunctionCounter.cpp
//       $(llvm-config --cxxflags --ldflags --libs) -o FunctionCounter.so
// 运行: clang -Xclang -load -Xclang ./FunctionCounter.so test.c

#include "llvm/Pass.h"
#include "llvm/IR/Function.h"
#include "llvm/IR/BasicBlock.h"
#include "llvm/IR/Instructions.h"
#include "llvm/Support/raw_ostream.h"
#include "llvm/IR/LegacyPassManager.h"
#include "llvm/Transforms/IPO/PassManagerBuilder.h"

using namespace llvm;

namespace {

// Function Pass 定义：统计每个函数的指令数
struct FunctionCounter : public FunctionPass {
    static char ID;  // LLVM 用 ID 唯一标识每个 Pass
    FunctionCounter() : FunctionPass(ID) {}

    // 每个函数调用一次，返回 true 表示修改了 IR
    bool runOnFunction(Function &F) override {
        int instCount = 0;

        // 遍历函数中所有基本块
        for (BasicBlock &BB : F)
            // 遍历基本块中所有指令
            for (Instruction &I : BB)
                instCount++;

        errs() << "函数 " << F.getName() << " 包含 " << instCount << " 条指令\n";
        return false;  // 未修改 IR（纯分析型 Pass）
    }
};

// 注册 Pass
char FunctionCounter::ID = 0;
static RegisterPass<FunctionCounter> X("func-counter",
    "统计每个函数包含的指令数量", false, false);

// 自动注册到 PassManager
static RegisterStandardPasses Y(
    PassManagerBuilder::EP_EarlyAsPossible,
    [](const PassManagerBuilder &Builder, legacy::PassManagerBase &PM) {
        PM.add(new FunctionCounter());
    });

} // anonymous namespace

语法规则：LLVM Pass 的结构继承模式

struct MyPass : public FunctionPass {
    static char ID;
    MyPass() : FunctionPass(ID) {}
    bool runOnFunction(Function &F) override {
        // 分析/转换 F 的 IR
        return false;  // false = 未修改
    }
};
char MyPass::ID = 0;
static RegisterPass<MyPass> X("mypass", "描述", false, false);

举一反三：LLVM 的 Pass 系统是编译器插件化的核心。每种 Pass 类型（Function/Module/Loop）对应不同粒度的 IR 遍历。runOnFunction 在每个函数上调用一次，返回 true 时 PassManager 知道 IR 变了需要重跑分析。

语法规则：LLVM IR 层级遍历（Function -> BasicBlock -> Instruction）

for (Function &F : M)           // 遍历模块中所有函数
    for (BasicBlock &BB : F)      // 遍历函数中所有基本块
        for (Instruction &I : BB) // 遍历基本块中所有指令
            /* 处理 I */

举一反三：LLVM IR 的层级结构：Module > Function > BasicBlock > Instruction。每条指令有 opcode（add/sub/call/br 等）和 operand（操作数）。这是编写编译器优化、静态分析工具、代码插桩的基础。

语法规则：RegisterStandardPasses 自动注入 Pass

static RegisterStandardPasses Y(
    PassManagerBuilder::EP_EarlyAsPossible,
    [](const PassManagerBuilder &B, legacy::PassManagerBase &PM) {
        PM.add(new MyPass());
    });

举一反三：RegisterStandardPasses 让 Pass 在编译时自动执行，无需 -load 显式指定。ExtensionPoint 决定 Pass 的插入位置：EP_EarlyAsPossible（最早）、EP_OptimizerLast（优化最后）等。
