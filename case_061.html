案例 61：线程池实现

阶段：工程能力锻造难度：⭐⭐⭐⭐练什么：并发核心能力、任务队列
手写一个线程池：固定数量的工作线程 + 任务队列 + 优雅关闭。这是 C++ 并发编程的必修课，面试高频考点。

📖 语法规则：std::thread 基础
std::thread t(func, args...); 创建线程并立即执行 func

示例：

std::thread t([] { cout << "Hello"; });
t.join();

💡 举一反三：

  • std::thread 对象必须 join() 或 detach()，否则析构时 terminate
  • 传参给线程函数：std::thread t(func, arg1, arg2); 参数按值拷贝
  • 用 lambda 是最常见的线程写法，注意捕获引用时的生命周期
📖 语法规则：std::condition_variable
cv.wait(lock, predicate); // 阻塞直到 predicate 为 true

示例：

std::unique_lock lk(mtx);
cv.wait(lk, []{ return ready; });
cv.notify_one();  // 唤醒一个等待线程
cv.notify_all();  // 唤醒所有

💡 举一反三：

  • wait 前必须持有 unique_lock，不能用 lock_guard
  • 推荐用带谓词的 wait：cv.wait(lock, []{ return condition; }); 等价于 while(!condition) cv.wait(lock);
  • notify 不需要持有锁，但不持有锁更快
📖 语法规则：RAII 关闭模式
析构函数中设置停止标志 → 唤醒所有线程 → join 等待

示例：

~ThreadPool() {
  { lock_guard lk(mtx); stop = true; }
  cv.notify_all();
  for(auto& t : workers) t.join();
}

💡 举一反三：

  • 停止标志要在锁内设置，避免竞态
  • 先设置标志再唤醒，保证线程看到标志后退出
  • join 放在析构函数外也行，但要保证 RAII
