案例 63：无锁队列（CAS 原子操作）

阶段：工程能力锻造难度：⭐⭐⭐⭐练什么：std::atomic、lock-free
基于 CAS（Compare-And-Swap）实现的单生产者单消费者无锁队列。没有 mutex，性能极高，是高性能系统的基石。

完整代码与逐行注释

case_063.cpp
1#include <iostream>


2#include <atomic>


💬 第2行：原子操作库


3#include <vector>


4#include <thread>


5 


6// 单生产者单消费者无锁队列（SPSC Lock-Free Queue）


7template<typename T, size_t Capacity = 1024>


💬 第7行：环形缓冲区，容量必须为 2 的幂


8class SPSCQueue {


9    static_assert((Capacity & (Capacity - 1)) == 0,


💬 第9行：编译期检查：容量是 2 的幂


10        "Capacity must be power of 2");


11 


12    T buffer[Capacity];


💬 第12行：环形缓冲区


13    // head：消费者读取位置；tail：生产者写入位置


14    alignas(64) std::atomic<size_t> head{0};


💬 第14行：alignas(64) 防止伪共享（false sharing）


15    alignas(64) std::atomic<size_t> tail{0};


💬 第15行：head 和 tail 各占独立缓存行


16 


17public:


18    bool push(const T& item) {


💬 第18行：生产者：入队


19        size_t currentTail = tail.load(std::memory_order_relaxed);


💬 第19行：relaxed 读，仅本线程修改 tail


20        size_t nextTail = (currentTail + 1) & (Capacity - 1);


💬 第20行：位与代替取模（2 的幂优化）


21        if (nextTail == head.load(std::memory_order_acquire)) {


💬 第21行：acquire 读 head，确保看到消费者的写


22            return false;  // 队列满


23        }


24        buffer[currentTail] = item;


💬 第24行：先写数据……


25        tail.store(nextTail, std::memory_order_release);


💬 第25行：再 release 更新 tail（消费者此时才能看到数据）


26        return true;


27    }


28 


29    bool pop(T& item) {


💬 第29行：消费者：出队


30        size_t currentHead = head.load(std::memory_order_relaxed);


31        if (currentHead == tail.load(std::memory_order_acquire)) {


💬 第31行：acquire 读 tail，确保看到生产者写入的数据


32            return false;  // 队列空


33        }


34        item = buffer[currentHead];


💬 第34行：读取数据


35        head.store((currentHead + 1) & (Capacity - 1),


36                   std::memory_order_release);


💬 第36行：release 更新 head


37        return true;


38    }


39};


40 


41int main() {


42    SPSCQueue<int, 64> queue;  // 64 个 int 的队列


43    std::atomic<bool> done{false};


44    std::atomic<int> sum{0};


💬 第44行：消费者累加结果


45 


46    // 生产者线程


47    std::thread producer([&] {


48        for (int i = 0; i < 100; ++i) {


49            while (!queue.push(i)) {


💬 第49行：自旋等待直到入队成功


50                std::this_thread::yield();


💬 第50行：yield 让出 CPU 时间片


51            }


52        }


53        done.store(true, std::memory_order_release);


54    });


55 


56    // 消费者线程


57    std::thread consumer([&] {


58        int val;


59        while (!done.load(std::memory_order_acquire) || queue.pop(val)) {


60            if (queue.pop(val)) {


61                sum.fetch_add(val);


💬 第61行：原子累加


62            }


63        }


64    });


65 


66    producer.join();


67    consumer.join();


68    std::cout << "Sum = " << sum.load() << std::endl;


💬 第68行：期望 4950（0~99 之和）


69    return 0;


70}
📖 语法规则：std::atomic
std::atomic var; 原子类型，所有操作不可分割

示例：

std::atomic counter(0);
counter.fetch_add(1);       // 原子自增
counter.store(42);           // 原子写入
int v = counter.load();      // 原子读取
counter.compare_exchange_weak(expected, desired);

💡 举一反三：

  • atomic 保证单个操作原子性，但不保证多个操作的组合原子性
  • 内存序默认是最强的 seq_cst，多数场景用 acquire/release 即可
  • atomic 可以用 flags 实现自旋锁
📖 语法规则：CAS 循环（compare_exchange_weak）
while (!atomic.compare_exchange_weak(expected, desired)) { expected = old; }

示例：

int expected = head.load();
while (!head.compare_exchange_weak(expected, newVal)) {
  // expected 已被更新为 head 当前值
}

💡 举一反三：

  • compare_exchange_weak 可能伪失败（spurious failure），必须在循环中使用
  • compare_exchange_strong 没有伪失败但性能略低
  • CAS 是无锁编程的核心原语
📖 语法规则：内存序 memory_order
store/load 可指定 memory_order（acquire/release/relaxed 等）

示例：

data.store(42, std::memory_order_release);  // 在此之前的所有写对后续 acquire 可见
int v = data.load(std::memory_order_acquire);  // 看到 release 之前的所有写入

💡 举一反三：

  • acquire-release 配对用于生产者-消费者模式
  • relaxed 只保证原子性不保证顺序，最快但慎用
  • 单消费者单生产者场景：tail 用 release 写入，head 用 acquire 读取
