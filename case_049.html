新语法精讲

XOR 加密原理与可逆性

语法规则：XOR（异或）加密的特性：明文 XOR 密钥 = 密文，密文 XOR 同一密钥 = 明文。即 加密和解密是完全相同的操作。这是因为 (A ^ K) ^ K = A。XOR 加密简单快速，但安全性低，仅适合非关键数据的混淆。
uint8_t plain  = 'A';      // 0x41
uint8_t key    = 'K';      // 0x4B
uint8_t cipher = plain ^ key; // 0x0A
uint8_t decoded = cipher ^ key; // 0x41 = 'A'（恢复原文）

istreambuf_iterator：一行读取整个文件

语法规则：istreambuf_iterator 是从 stream buffer 读取字符的迭代器。用两个参数构造 vector 时，第一个是开始位置，默认构造的第二个参数代表 EOF。这是 C++ 中读取整个文件的最短写法，效率接近 mmap。
// 一行读完整个文件
ifstream f("data.bin", ios::binary);
vector data((istreambuf_iterator(f)),
                     istreambuf_iterator());
// 等效于
string content((istreambuf_iterator(f)),
               istreambuf_iterator());

ofstream::write 写入原始字节

语法规则：write(const char* s, streamsize n) 将字节缓冲区直接写入文件，不进行任何格式化转换。第一个参数需从 uint8_t* 转为 char*（reinterpret_cast），第二个参数是要写入的字节数。适合二进制数据。
vector buf = {0x48, 0x65, 0x6C, 0x6C, 0x6F};  // "Hello"
ofstream f("out.bin", ios::binary);
f.write(reinterpret_cast(buf.data()), buf.size());

举一反三

  • 使用 AES 对称加密（更安全）：
    #include 
AES_KEY encKey;
AES_set_encrypt_key((uint8_t*)key.c_str(), 128, &encKey);
AES_encrypt(input, output, &encKey);

    XOR 加密极易被破解。生产环境应使用 AES-256-GCM 等标准对称加密算法，并配合随机 IV。

  • 数据完整性校验：HMAC 防篡改：
    #include 
HMAC(EVP_sha256(), key.c_str(), key.size(),
     data.data(), data.size(), digest, &len);

    加密不能防止篡改。HMAC（哈希消息认证码）可以验证数据是否被修改过，加密和认证应配合使用。
