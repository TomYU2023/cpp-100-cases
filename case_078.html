案例 78：简易性能剖析器

用 RAII + chrono 实现轻量级函数性能统计，支持按总耗时排序报告，堪比 Python cProfile。

核心知识点：RAII、性能剖析、chrono 高精度计时、structured binding C++17、iomanip 格式化

完整代码（含逐行注释）

// 案例 78：简易性能剖析器（Profiler）
// 使用 RAII 和 chrono 实现一个函数级性能计时器，类似 Python 的 cProfile。

#include <iostream>
#include <string>
#include <vector>
#include <chrono>
#include <unordered_map>
#include <algorithm>
#include <iomanip>

/* 语法规则：性能剖析（Profiling）
   Profiling 回答三个问题：
   1. 哪个函数最耗时？（热点分析）
   2. 函数被调用了多少次？
   3. 每次调用平均耗时多少？

   常用工具：
   - perf (Linux): perf record ./app && perf report
   - Instruments (macOS)
   - Visual Studio Profiler (Windows)
   - Valgrind/Callgrind (Linux)
   本案例实现的是一个轻量级代码内嵌 Profiler，适合快速定位瓶颈。

   计时原理：
     auto start = high_resolution_clock::now();
     // ... 被测代码 ...
     auto end = high_resolution_clock::now();
     auto duration = end - start;

   举一反三：生产环境常用的 Profiler（如 Tracy、Optick）支持
   多线程、GPU、网络、内存等维度的可视化分析，比本案例强大得多，
   但核心思想相同：记录时间戳 + 统计聚合。
*/

// RAII 计时器：构造时开始计时，析构时记录结果
class ScopedTimer {
    using Clock = std::chrono::high_resolution_clock;
    using TimePoint = Clock::time_point;

    std::string funcName;
    TimePoint start;

    // 全局统计表：函数名 → {总耗时, 调用次数}
    static std::unordered_map<std::string,
        std::pair<long long, int>>& stats() {
        static std::unordered_map<std::string, std::pair<long long, int>> s;
        return s;
    }

public:
    explicit ScopedTimer(std::string name)
        : funcName(std::move(name)), start(Clock::now()) {}

    ~ScopedTimer() {
        auto end = Clock::now();
        auto duration = std::chrono::duration_cast<
            std::chrono::microseconds>(end - start).count();  // 微秒

        auto& [total, count] = stats()[funcName];
        total += duration;
        count += 1;
    }

    // 打印统计报告
    static void report() {
        // 转为 vector 以排序
        std::vector<std::tuple<std::string, long long, int>> entries;
        for (auto& [name, pair] : stats()) {
            entries.emplace_back(name, pair.first, pair.second);
        }

        // 按总耗时降序排序
        std::sort(entries.begin(), entries.end(),
            [](auto& a, auto& b) { return std::get<1>(a) > std::get<1>(b); });

        std::cout << "\n========== 性能分析报告 ==========" << std::endl;
        std::cout << std::left << std::setw(20) << "函数名"
                  << std::right << std::setw(12) << "总耗时(μs)"
                  << std::setw(10) << "调用次数"
                  << std::setw(12) << "平均(μs)" << std::endl;
        std::cout << std::string(54, '-') << std::endl;

        for (auto& [name, total, count] : entries) {
            std::cout << std::left << std::setw(20) << name
                      << std::right << std::setw(12) << total
                      << std::setw(10) << count
                      << std::setw(12) << (total / count) << std::endl;
        }
        std::cout << "=====================================" << std::endl;
    }
};

// 配合宏使用更简洁
#define PROFILE_SCOPE(name) ScopedTimer _timer_##__LINE__(name)

// 假装是业务代码
void heavyCalculation() {
    PROFILE_SCOPE("heavyCalculation");
    long long sum = 0;
    for (int i = 0; i < 1000000; ++i) sum += i;  // 模拟计算
}

void lightWork() {
    PROFILE_SCOPE("lightWork");
    volatile int x = 42; (void)x;
}

void mixedWork() {
    PROFILE_SCOPE("mixedWork");
    heavyCalculation();
    lightWork();
    heavyCalculation();  // 调用两次
}

int main() {
    std::cout << "运行性能剖析..." << std::endl;

    mixedWork();
    heavyCalculation();
    lightWork();

    ScopedTimer::report();
    return 0;
}

语法规则：RAII（资源获取即初始化）

// RAII：将资源生命周期绑定到对象生命周期
class FileGuard {
  FILE* f;
public:
  FileGuard(const char* path) : f(fopen(path, "r")) {}
  ~FileGuard() { if (f) fclose(f); }  // 离开作用域自动关闭
};
// C++ 标准库中大量使用：unique_ptr, lock_guard, ifstream 等

💡 举一反三：RAII 是 C++ 最重要的惯用法，没有之一。它解决了其他语言需要 try-finally 或 defer 才能解决的问题。任何时候你需要"获取-使用-释放"模式，就写一个 RAII 类。

语法规则：结构体绑定（C++17）与 auto& [a, b]

std::pair<int, std::string> p = {1, "hello"};
auto& [id, name] = p;  // id=1, name="hello"
// 也可以用 auto [id, name] = p; 获取副本

💡 举一反三：C++17 结构化绑定让 pair/tuple 解包变得优雅。注意 & 表示引用，不加 & 是副本。const auto& 表示只读引用。

语法规则：std::setw + std::left/right 格式化输出

std::cout << std::left << std::setw(20) << "Name"
          << std::right << std::setw(10) << 12345;
// 输出: Name                    12345

💡 举一反三： 提供了 setw/setfill/setprecision/fixed 等操纵符。setw 只对下一个输出有效，其他操纵符是持久的。
