案例 8：文本字符/单词频率统计

读取一段文本，统计每个字符和每个单词的出现频率，练习 map 和 stringstream 的使用。

核心知识点：map 统计、stringstream 分词、const 引用

完整代码（含逐行注释）

// 案例 8：文本字符 / 单词频率统计
// 使用 map 统计频率
#include <iostream>
#include <string>
#include <map>
#include <sstream>   // stringstream 用于分词
using namespace std;

int main() {
    string text = "hello world hello cpp world cpp cpp";

    // 1. 字符频率统计
    map<char, int> charCount;
    for (char c : text) {
        if (c != ' ') {           // 忽略空格
            charCount[c]++;       // 自动初始化为 0 并 +1
        }
    }

    cout << "=== 字符频率 ===" << endl;
    for (const auto& pair : charCount) {
        cout << "'" << pair.first << "': " << pair.second << endl;
    }

    // 2. 单词频率统计
    map<string, int> wordCount;
    stringstream ss(text);
    string word;
    while (ss >> word) {          // 按空白字符分词
        wordCount[word]++;
    }

    cout << "\n=== 单词频率 ===" << endl;
    for (const auto& pair : wordCount) {
        cout << pair.first << ": " << pair.second << endl;
    }

    return 0;
}

map 统计频率

【规则】std::map 结合 ++ 操作可实现简洁的频率统计——map[key] 在 key 不存在时自动插入默认值 0 并返回引用。适用于词频统计、投票计数、标签分布等。

【原理】map 内部是红黑树，每次 [] 操作 O(log n)。map[key] 的行为有微妙之处：如果 key 不存在，它会在 map 中插入一个值为 0 的条目（值类型的默认构造）。这就是 ++map[key] 能工作的原因——即使 key 从未见过，也会先插入 0 再递增。如果只需要查询而不插入，应该用 find() 或 C++20 的 contains()。

【常见陷阱】1. 使用 map.count(key) 判断存在性——count 在有重复键时返回更高值，对 map（键唯一）等价于 contains() 但要遍历所有匹配；2. 在 const 成员函数中使用 map[key]——operator[] 不是 const，因为找不到时会插入，应改用 at() 或 find()。

#include <map>
map<string, int> freq;
freq[word]++; // 键不存在时自动插入并初始化为 0

【变体练习一】将当前程序的 map 替换为 unordered_map，重新编译运行。对比两种容器在 100000 个单词统计场景下的运行时间。

【变体练习二】实现「Top-K 高频词」功能：在统计完所有词频后，使用 partial_sort 或 nth_element 只提取频率最高的 10 个词，而不对全部词排序。

【延伸阅读】搜索「C++ map vs unordered_map performance benchmark」；查阅 C++20 的 std::map::contains() 和 std::erase_if；参考案例 022（map 电话号码簿）。

stringstream 分词

【规则】std::istringstream 将字符串当作输入流处理，用 operator>> 可自然按空格切分词。适用于文本解析、数据分词、配置读取等场景。

【原理】stringstream 继承自 istream，其 operator>> 对 string 类型的重载会自动跳过前导空白并以空白字符为分隔符——与 cin >> 行为一致。如果要按其他分隔符（如逗号、冒号）分词，需要 getline(ss, token, delimiter)。它的底层是 std::stringbuf，所有数据在内存中操作，无磁盘 I/O。

【常见陷阱】1. 分词后 stringstream 的内部读取位置在末尾，忘了调用 ss.clear() 和 ss.str() 重置就把新字符串灌进去——读取位置仍在末尾，导致后续读取为空；2. >> 操作会把连续的空白压缩为单个分隔，如果要保留空字段（如 CSV 中两个连续逗号表示的空列），必须改用 getline。

#include <sstream>
stringstream ss(text);
string word;
while (ss >> word) { /* 处理 */ }

【变体练习一】改用 getline(ss, word, ' ') 按空格分词，然后再用 regex 移除标点符号（如句号、逗号、引号），对比与 operator>> 分词的结果差异。

【变体练习二】读取一个多行段落，先用 stringstream 按行分割（getline），再对每行使用另一个 stringstream 分词，构建「行号 → 单词列表」的二维结构。

【延伸阅读】搜索「C++ stringstream performance」和「how to reset stringstream」；查阅 C++23 的 std::views::split 作为新式分词方案；参考案例 047（CSV 解析器）。

基于范围的 for + const auto&

【规则】for(const auto& item : container) 以只读引用方式遍历容器，避免拷贝开销。适用于统计、查找、输出等不需要修改元素的只读遍历。

【原理】const auto& 告诉编译器两件事：const 禁止修改（防止意外改动），& 按引用传递（避免拷贝）。当元素是 std::string 或自定义大对象时，拷贝开销可能是基准性能的 100 倍以上。编译器看到 const 引用后可以做更多优化——它知道被引用对象在循环体内不会改变，可以安全地缓存成员访问结果。

【常见陷阱】1. 写了 for(const auto& p : map)，但在循环体内访问 p.first 和 p.second——这没错，但更推荐结构化绑定 for(const auto& [k, v] : map)，可读性更好；2. 想在 for 循环体中修改元素但写了 const auto&——编译报错，应改为 auto&。

for (const auto& pair : wordCount) {
    cout << pair.first << ": " << pair.second << endl;
}
