新语法精讲

递归下降解析器

语法规则：每个语法结构对应一个解析函数，函数间互相递归调用形成调用链。核心原则：每个函数解析成功后，pos 必须指向下一个待解析位置；解析失败则抛异常。这是最直观的手写解析器模式，适合 JSON / 表达式等上下文无关语法。
// 解析器调用链
parseValue()
  ├─ parseObject()
  │   ├─ parseString()  // 解析 key
  │   └─ parseValue()   // 递归解析 value
  └─ parseArray()
      └─ parseValue()   // 递归解析元素

std::variant（C++17 类型安全联合体）

语法规则：variant 可以安全地存储多种类型中的一种，比 C 语言 union 更安全（自动调用析构、类型检查）。用 get<T>(v) 提取值，用 holds_alternative<T>(v) 检查类型。
variant v;
v = 42;              // 存 int
int x = get(v); // 取出 int
v = "hello";         // 存 string
// int y = get(v);  // 抛异常！当前存的是 string
if (holds_alternative(v)) {
    cout << get(v);
}

R 原始字符串字面量（C++11）

语法规则：R 前缀创建原始字符串，反斜杠不再是转义符。语法：R"delim(内容)delim"。delim 是可选的定界符。在 JSON、正则表达式、文件路径等含大量反斜杠的场景极为实用。
// 普通字符串：需要转义
string s1 = "C:\\Users\\Admin\\file.txt";
// 原始字符串：原样书写
string s2 = R"(C:\Users\Admin\file.txt)";
// 带定界符（内容含括号时）
string s3 = R"delim({"key": [1, 2]})delim";

举一反三

  • 使用 nlohmann/json 库（工业标准）：
    #include 
using json = nlohmann::json;
json j = json::parse(R"({"name":"Alice"})");
cout << j["name"];

    nlohmann/json 是 C++ 中最流行的 JSON 库，header-only，一行安装。支持序列化/反序列化、类 JSON 字面量语法。

  • 支持序列化：将 JsonValue 转回 JSON 字符串：
    void printJson(const JsonValue& v) {
    visit([](auto&& arg) {
        using T = decay_t;
        if constexpr (is_same_v) cout << '"' << arg << '"';
        else if constexpr (is_same_v) cout << "null";
        else cout << arg;
    }, v);
}

    利用 std::visit + if constexpr 实现打印，这是 C++17 处理 variant 的标准模式，避免了冗长的 if/else 类型判断。
