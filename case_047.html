新语法精讲

单例模式 + static 局部变量（Meyer's Singleton）

语法规则：在函数内声明 static 局部变量，C++11 起保证线程安全初始化（编译器自动添加双重检查锁定）。这是 C++ 中最简洁可靠的单例实现，无需手动 new/delete，生命周期随程序结束自动析构。
class Singleton {
    Singleton() {}  // 私有构造
public:
    Singleton(const Singleton&) = delete;  // 禁止拷贝
    static Singleton& get() {
        static Singleton instance;  // 只构造一次
        return instance;
    }
};

= delete 禁用函数

语法规则：= delete 明确禁用某个函数。相比传统的私有声明，= delete 在编译期就报错（而非链接期），错误信息更清晰，且对友元和成员函数同样生效。
class NonCopyable {
public:
    NonCopyable() = default;           // 使用默认构造
    NonCopyable(const NonCopyable&) = delete;  // 禁止拷贝
    NonCopyable& operator=(const NonCopyable&) = delete;
};
NonCopyable a;
// NonCopyable b = a;  // 编译错误！

std::lock_guard 与 RAII 加锁

语法规则：lock_guard 在构造时加锁（mutex.lock()），析构时自动解锁。即使函数中途 return 或抛异常，也能保证锁被释放——这是 RAII 的经典应用。
mutex mtx;
void safeFunction() {
    lock_guard lock(mtx);  // 加锁
    // 临界区代码
}  // lock 析构，自动解锁
// 替代：手动管理
mtx.lock();
// 如果这里抛异常，mtx.unlock() 永远不会执行！
mtx.unlock();

举一反三

  • 使用 spdlog 工业级日志库：
    #include 
spdlog::info("Hello {}!", "World");
// 同时输出到文件和控制台
spdlog::set_pattern("[%Y-%m-%d %H:%M:%S.%e] [%l] %v");

    spdlog 是 C++ 最快的日志库之一，header-only，支持异步日志、多 sink、自定义格式、彩色输出等，工业项目直接用它。

  • 按日期轮转日志文件 + 条件编译关闭 Debug：
    #ifdef NDEBUG
#define LOG_DEBUG(msg) ((void)0)  // Release 模式下编译为空
#else
#define LOG_DEBUG(msg) Logger::getInstance().debug(msg)
#endif

    用预处理器宏实现 Debug 日志在 Release 版本中零开销。文件轮转（RotatingFileSink）可以限制日志文件大小和数量。
