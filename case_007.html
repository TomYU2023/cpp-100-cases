案例 7：字符串反转/查找/替换

实现字符串的三种基本操作：反转、子串查找和替换，练习 string 类的常用方法。

核心知识点：string 方法、reverse 算法、find 和 replace

完整代码（含逐行注释）

// 案例 7：字符串反转 / 查找 / 替换
// 练习 std::string 的核心操作
#include <iostream>
#include <string>
#include <algorithm> // reverse()
using namespace std;

// 反转字符串
string reverseStr(const string& s) {
    string result = s;
    reverse(result.begin(), result.end());
    return result;
}

// 查找并替换所有匹配的子串
string replaceAll(string s, const string& from, const string& to) {
    size_t pos = 0;
    while ((pos = s.find(from, pos)) != string::npos) {
        s.replace(pos, from.length(), to); // 替换
        pos += to.length();                // 跳过已替换部分
    }
    return s;
}

int main() {
    string text = "Hello World, Hello C++";

    // 1. 反转
    cout << "反转: " << reverseStr(text) << endl;

    // 2. 查找
    size_t pos = text.find("World");
    if (pos != string::npos) {
        cout << "\"World\" 出现在位置 " << pos << endl;
    } else {
        cout << "未找到" << endl;
    }

    // 3. 替换
    string result = replaceAll(text, "Hello", "你好");
    cout << "替换后: " << result << endl;

    return 0;
}

运行结果

反转: ++C olleH ,dlroW olleH
"World" 出现在位置 6
替换后: 你好 World, 你好 C++

std::reverse

【规则】std::reverse(begin, end) 反转指定范围内元素的顺序，包含在 中。适用于字符串反转、数组逆序、回文检查等场景。

【原理】reverse 内部实现为逐对交换首尾元素（swap(first, last-1)），时间复杂度 O(n/2) 次交换。对于支持随机访问的迭代器（如 vector、string），复杂度为 O(n)。对于双向迭代器（如 list），每次定位需遍历，复杂度不变但常数更大。reverse 是纯算法函数，不关心容器类型。

【常见陷阱】1. 习惯写成 reverse(str) 而非 reverse(str.begin(), str.end())——reverse 是两个迭代器参数，不能直接传容器；2. 在 range-for 中调用 reverse 操作了同一个容器，导致迭代器失效。

#include <algorithm>
string s = "abc";
reverse(s.begin(), s.end()); // s = "cba"

【变体练习一】不用 std::reverse，手动实现 reverse_string 函数（使用双指针法，一头一尾向中间交换），并对比两种实现在大字符串（100 万字符）上的性能。

【变体练习二】实现「句子反转」：输入 'Hello World C++'，输出 'olleH dlroW ++C'（每个单词内部反转，单词顺序不变）。结合 find(' ') 和 reverse 完成。

string::find()

【规则】string::find(str, pos) 从 pos 开始搜索子串 str，返回首次出现的索引（size_t），未找到返回 string::npos。适用于文本查找、模式匹配、URL 解析等。

【原理】string::find 的具体实现取决于标准库——GCC 的 libstdc++ 使用两阶段算法：先用 memchr 快速跳过不匹配的字节，再用 memcmp 精确匹配；MSVC 使用带优化的朴素搜索。对于大型文本搜索，KMP 或 Boyer-Moore 算法更优。npos 是 size_t 的最大值（static const），不要与 int 混用。

【常见陷阱】1. 将 find 返回值赋给 int 变量——当返回 npos 时，-1 和 size_t 最大值之间的截断导致逻辑错误；2. 在循环中用 find 查找所有匹配时没有更新 pos = found + 1，导致死循环在同一个位置反复匹配。

size_t pos = s.find("hello");
if (pos != string::npos) { /* 找到了 */ }

【变体练习一】实现函数 vector find_all(const string& text, const string& pattern)，返回 pattern 在 text 中所有出现位置的列表。使用 while 循环和 find 的第二个参数实现。

【变体练习二】实现简易的文本搜索替换工具：读取文件，使用 find + replace 将所有「C++」替换为「C++ (Modern)」，写入新文件。

【延伸阅读】搜索「C++ string find complexity」了解不同实现的时间复杂度；查阅 Boyer-Moore 和 KMP 算法的 C++ 实现；参考案例 086（std::search 子序列查找）。

string::replace()

【规则】string::replace(pos, len, newStr) 将从 pos 开始的 len 个字符替换为 newStr。可删除、替换或插入内容。适用于文本修改、模板填充、数据清洗等。

【原理】replace 会修改原字符串，可能触发内存重分配（如果 newStr 比被替换部分长）。它是 O(n) 操作——被替换位置之后的字符都需要移动。批量替换时如果每次调用都触发整体位移，复杂度会达到 O(n·m)。更高效的做法是先遍历确定总位移量，一次性构造新字符串。

【常见陷阱】1. 在循环中调用 replace，但替换操作改变了字符串长度，导致后续位置的索引全部偏移——正确的做法是从后往前替换，或使用 find 的返回值作为相对位置；2. replace 的 pos 参数越界——如果 pos >= size() 或 pos + len > size()，会抛出 std::out_of_range。

s.replace(pos, len, "new");
// 从 pos 开始替换 len 个字符

【变体练习一】实现高效的「全部替换」函数 replace_all(string& s, const string& from, const string& to)，使用 while(find) + replace 循环，注意处理 to 包含 from 导致的死循环（如把 'a' 替换为 'aa' 时会无限增长）。

【变体练习二】实现一个简易 URL 编码器：扫描字符串，将空格替换为 %20，将特殊字符替换为 %XX（XX 为十六进制 ASCII 码）。使用 replace 和字符串拼接完成。

【延伸阅读】搜索「C++ replace_all string performance」了解批量替换的最佳实现；查阅 std::regex_replace 的使用方法（参考案例 036）。
