新语法精讲

main 函数参数 argc/argv

语法规则：argc（argument count）是参数个数，argv（argument vector）是参数字符串数组。argv[0] 始终是程序名本身，真正的参数从 argv[1] 开始。argv 中每个元素是 char*（C 风格字符串），需要时可以转为 std::string。
// 测试命令：
// program.exe --name=Alice --age=25 -v input.txt output.txt
// argv[0] = "program.exe"
// argv[1] = "--name=Alice"
// argv[2] = "--age=25"
// argv[3] = "-v"
// argv[4] = "input.txt"
// argv[5] = "output.txt"

std::string::starts_with（C++20）

语法规则：检查字符串是否以指定前缀开头。C++20 新增，之前需要用 compare(0, n, prefix) 或 rfind(prefix, 0) == 0 替代。同样还有 ends_with。
string s = "hello.cpp";
bool a = s.starts_with("he");   // true
bool b = s.ends_with(".cpp");    // true
bool c = s.ends_with(".h");      // false
// 等价写法（C++17）
bool d = s.rfind(".cpp") == s.size() - 5;  // true

std::string::find 和 std::string::npos

语法规则：find() 返回子串首次出现的位置（size_t 类型）。未找到时返回 std::string::npos，这是一个表示“无效位置”的静态常量，实际值为 -1 转为 size_t 的最大值。
string s = "key=value";
size_t pos = s.find('=');
if (pos != string::npos) {
    string k = s.substr(0, pos);   // "key"
    string v = s.substr(pos + 1);  // "value"
}

map::find 与 end() 用法

语法规则：map::find(key) 返回指向该键值对的迭代器。如果 key 不存在，返回 end()。这是检查 key 是否存在的最佳方式，比 count(key) 更语义清晰。
map m = {{"a",1},{"b",2}};
auto it = m.find("a");
if (it != m.end()) {
    cout << it->second;  // 1
} else {
    cout << "not found";
}

举一反三

  • 完整的 CLI 库（CLI11 / cxxopts）：
    #include "CLI/CLI.hpp"
CLI::App app{"My Tool"};
string filename;
app.add_option("-f,--file", filename, "Input file");
CLI11_PARSE(app, argc, argv);

    工业级项目推荐使用 CLI11 或 cxxopts 等 header-only 库，它们处理了长短选项、帮助生成、类型验证、默认值等复杂逻辑。

  • Windows 下的宽字符参数：
    int wmain(int argc, wchar_t* argv[]) {
    wstring arg = argv[1];
}

    Windows 支持宽字符版本 wmain，参数为 wchar_t*，适合处理 Unicode 路径和中文参数。
