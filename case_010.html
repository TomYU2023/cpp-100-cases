案例 10：复数类(运算符重载)

定义一个复数类，重载 +、-、*、<< 等运算符，深入理解运算符重载的语法和应用。

核心知识点：运算符重载、友元函数、const 成员函数

完整代码（含逐行注释）

// 案例 10：复数类（运算符重载）
// 重载 +、-、*、==、<< 运算符
#include <iostream>
using namespace std;

class Complex {
private:
    double real; // 实部
    double imag; // 虚部

public:
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // 成员函数重载 + 运算符
    Complex operator+(const Complex& other) const {
        return Complex(real + other.real, imag + other.imag);
    }

    // 成员函数重载 - 运算符
    Complex operator-(const Complex& other) const {
        return Complex(real - other.real, imag - other.imag);
    }

    // 成员函数重载 * 运算符：(a+bi)(c+di) = (ac-bd)+(ad+bc)i
    Complex operator*(const Complex& other) const {
        return Complex(
            real * other.real - imag * other.imag,
            real * other.imag + imag * other.real
        );
    }

    // 重载 == 运算符
    bool operator==(const Complex& other) const {
        return real == other.real && imag == other.imag;
    }

    // 友元函数重载 << 输出运算符
    friend ostream& operator<<(ostream& os, const Complex& c) {
        os << c.real;
        if (c.imag >= 0) os << " + " << c.imag << "i";
        else             os << " - " << -c.imag << "i";
        return os;
    }
};

int main() {
    Complex a(3, 2);  // 3 + 2i
    Complex b(1, 4);  // 1 + 4i

    Complex sum = a + b;    // 调用 operator+
    Complex diff = a - b;   // 调用 operator-
    Complex prod = a * b;   // 调用 operator*

    cout << "a = " << a << endl;
    cout << "b = " << b << endl;
    cout << "a + b = " << sum << endl;
    cout << "a - b = " << diff << endl;
    cout << "a * b = " << prod << endl;
    cout << "a == b ? " << (a == b ? "true" : "false") << endl;

    return 0;
}

运行结果

a = 3 + 2i
b = 1 + 4i
a + b = 4 + 6i
a - b = 2 - 2i
a * b = -5 + 14i
a == b ? false

成员函数重载二元运算符

【规则】ReturnType operator@(const ClassName& rhs) const 以成员函数形式重载二元运算符，左操作数是 this 指向的对象。适用于 +、-、*、/、== 等运算。

【原理】成员函数重载时只有右操作数可以隐式类型转换——因为左操作数必须是指定类型的对象才能触发成员函数查找。如果希望两边都可以隐式转换（如 Complex + double 和 double + Complex 都成立），需要对称地提供友元函数版本。编译器选择规则：有成员函数优先用成员，否则找友元/自由函数。

【常见陷阱】1. 重载 operator+ 时修改了 this 对象——operator+ 应该是 const 成员函数，返回新对象，不应有副作用；如果要修改自身，应重载 operator+=；2. 在 operator+ 内返回了局部变量的引用——悬垂引用是未定义行为，应返回值（非引用）。

Complex operator+(const Complex& other) const {
    return Complex(real + other.real, imag + other.imag);
}

【变体练习一】为案例 010 的 Complex 类新增 operator-、operator*、operator/，分别实现复数减法、乘法和除法。注意除法的分母处理。

【变体练习二】实现 Complex 与 double 的混合运算：重载 friend Complex operator+(double d, const Complex& c)，使得 3.0 + c1 也能正常工作。然后测试 c1 + 3.0 + c2 的表达式树生成。

友元函数重载 <<

【规则】friend ostream& operator<<(ostream& os, const ClassName& obj) 使自定义类型支持 cout 输出。因左操作数是 ostream 而非自定义类型，必须用友元而非成员函数实现。

【原理】operator<< 的左操作数必须是 ostream&（输出流对象），而 ostream 是标准库类型，不能为其添加成员函数。友元函数虽然写在类定义内部，但本质是全局函数，只是被授予了访问 private 成员的权限。返回 ostream& 是为了支持链式调用：cout << a << b << c。

【常见陷阱】1. 忘记在 operator<< 声明前加 friend 关键字——没有 friend 则变成普通全局函数，无法访问私有成员；2. 函数内修改了传入对象的内部状态——operator<< 应该只读，不应修改 obj，参数应声明为 const ClassName&。

friend ostream& operator<<(ostream& os, const Complex& c) {
    os << c.real << " + " << c.imag << "i";
    return os;
}

【变体练习一】为案例 010 的 Complex 类重载 operator>>，使其支持从 cin 读取复数（格式如 '3.0+4.0i' 或 '3.0 4.0'）。在实现中处理用户输入的各种边界情况。

【变体练习二】重载 operator<< 使 Complex 输出更美观——当虚部为 0 时只输出实部，虚部为负时输出 '3-4i' 而非 '3+-4i'。这需要特殊的格式化逻辑。

默认参数构造函数

【规则】构造函数参数可指定默认值，允许使用不同数量参数构造对象。ClassName(double r = 0, double i = 0) 可作无参/单参/双参构造。适用于需要灵活初始化的类。

【原理】默认参数本质是编译期展开——编译器在调用处将缺失的实参替换为默认值，与函数重载等价但更简洁。但默认参数也有局限：不能跳过中间参数（如 f(a, , c) 无效），且默认值只能出现在声明中（定义中不能重复）。默认参数和函数重载同时出现可能导致二义性。

【常见陷阱】1. 同时在声明和定义中写了默认参数值——编译错误（重复默认参数），只在声明处写一次即可；2. 一个构造函数有默认参数（ClassName(int a=0)），另一个构造函数签名类似（ClassName()），当调用无参构造时编译器报 ambiguous。

Complex(double r = 0, double i = 0) : real(r), imag(i) {}

【变体练习一】为案例 010 的 Complex 类添加全默认参数构造函数 Complex(double r=0, double i=0)，然后删除默认构造函数和无参构造，测试三种构造方式（Complex()、Complex(3)、Complex(3,4)）是否都正常。

【变体练习二】使用 C++11 的委托构造函数改写 Complex：Complex(double r, double i) : real(r), imag(i) {}，然后用 Complex() : Complex(0, 0) {} 委托。对比默认参数和委托构造两种风格的优劣。
