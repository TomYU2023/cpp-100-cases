案例 73：std::clamp 值裁剪

使用 clamp 将值限制在指定范围内，结合其他数值算法处理数据。

核心知识点：clamp、min/max、数值范围限制

完整代码（含逐行注释）

// 案例 73：std::clamp 值裁剪
// 将值限制在最小和最大边界内
#include <iostream>
#include <algorithm>
#include <vector>
#include <string>
using namespace std;

int main() {
    // 1. 基本用法：将值裁剪到 [min, max]
    cout << "=== std::clamp ===" << endl;
    cout << "clamp(5, 0, 10) = " << clamp(5, 0, 10) << endl;   // 5
    cout << "clamp(-3, 0, 10) = " << clamp(-3, 0, 10) << endl; // 0
    cout << "clamp(15, 0, 10) = " << clamp(15, 0, 10) << endl; // 10

    // 2. 裁剪容器中所有值
    vector<int> values = {-5, 0, 25, 50, 75, 100, 125};
    cout << "\n原始数据: ";
    for (int v : values) cout << v << " ";
    cout << endl;

    // 裁剪到 [0, 100]
    cout << "裁剪到 [0, 100]: ";
    for (int v : values) {
        cout << clamp(v, 0, 100) << " ";
    }
    cout << endl;

    // 3. 用 clamp 实现进度条百分比
    cout << "\n=== 进度百分比 ===" << endl;
    vector<double> rawPercents = {-10.0, 0.0, 45.5, 89.3, 100.0, 120.0, 55.0};

    for (double raw : rawPercents) {
        double percent = clamp(raw, 0.0, 100.0);
        int barWidth = static_cast<int>(percent / 5); // 每 5% 一个字符
        cout << raw << "%"
             << " -> " << string(barWidth, '#')
             << " " << percent << "%" << endl;
    }

    // 4. 对比 min/max 写法
    cout << "\n=== clamp vs min+max ===" << endl;
    int val = 150;
    cout << "clamp(150, 0, 100) = " << clamp(val, 0, 100) << endl;
    cout << "min(max(150,0),100) = " << min(max(val, 0), 100) << endl;

    return 0;
}

语法规则：std::clamp 基本用法

#include <algorithm>
int result = clamp(value, min, max);
// 等价于 max(min, min(max, value))
// 但只调用一次比较（更高效）

举一反三：clamp (C++17) 比嵌套 min/max 更清晰且可能更高效。对自定义类型，要求 operator< 已定义。clamp 的参数顺序：值、下界、上界。

语法规则：clamp 与容器操作

vector<int> v;
transform(v.begin(), v.end(), v.begin(),
    [](int x) { return clamp(x, 0, 100); });
// 批量裁剪数据到合法范围

举一反三：clamp 配合 transform 是批量数据清洗的常见组合。也可用 ranges (C++20)：v | views::transform([](int x){ return clamp(x, 0, 100); })。

语法规则：clamp 的注意事项

// clamp 对下界大于上界的行为是未定义的
clamp(5, 10, 0); // UB!
// 如果使用自定义比较器，需保证 lo <= hi
// 断言：assert(lo <= hi);

举一反三：clamp 假定 lo <= hi，否则行为未定义。对于浮点数 NaN 的比较，clamp 的结果也是未定义的。可以写一个 safe_clamp 先检查参数有效性。
