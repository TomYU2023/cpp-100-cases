新语法精讲

std::thread::hardware_concurrency() 获取 CPU 核心数

语法规则：返回系统支持的并发线程数（通常为 CPU 核心数）。返回 0 时表示无法确定，应用代码应提供回退值。这是确定线程池大小的推荐方法。
unsigned int n = thread::hardware_concurrency();
cout << "Your CPU supports " << n << " concurrent threads.";
// 常用公式
size_t optimal = max(1u, n - 1);  // 留一个核给系统

fs::file_size 和 fs::exists

语法规则：std::filesystem（C++17）提供跨平台的文件系统操作。exists(path) 检查文件是否存在，file_size(path) 返回文件字节数。注意 file_size 对不存在或无法访问的文件会抛异常。
namespace fs = filesystem;
try {
    if (fs::exists("config.ini")) {
        auto sz = fs::file_size("config.ini");
        cout << sz << " bytes";
    }
} catch (fs::filesystem_error& e) {
    cerr << e.what();
}

vector::emplace_back 就地构造

语法规则：emplace_back(args...) 直接在 vector 尾部构造元素，无需临时对象拷贝。相比 push_back（先构造临时对象再拷贝/移动），emplace_back 性能更好，尤其对大对象和不可拷贝类型。
struct Point { int x, y; Point(int a, int b): x(a), y(b) {} };
vector pts;
pts.emplace_back(1, 2);  // 直接构造！
// 等效的 push_back 写法：
pts.push_back(Point(1, 2));  // 先构造临时对象，再拷贝
