案例 51：策略模式

实现策略模式，将排序算法封装为可替换的策略对象，理解行为参数化。

核心知识点：策略模式、函数对象、模板策略

完整代码（含逐行注释）

// 案例 51：策略模式
// 将排序策略封装为可替换的函数对象
#include <iostream>
#include <vector>
#include <algorithm>
#include <string>
#include <functional>
using namespace std;

// 策略接口：排序策略
template <typename T>
using SortStrategy = function<void(vector<T>&)>;

// 具体策略：升序排序
template <typename T>
void ascendingSort(vector<T>& data) {
    sort(data.begin(), data.end());
}

// 具体策略：降序排序
template <typename T>
void descendingSort(vector<T>& data) {
    sort(data.begin(), data.end(), greater<T>());
}

// 具体策略：按绝对值排序（数值类型）
void absoluteSort(vector<int>& data) {
    sort(data.begin(), data.end(),
         [](int a, int b) { return abs(a) < abs(b); });
}

// 上下文：使用策略的排序器
template <typename T>
class Sorter {
private:
    vector<T> data;
    SortStrategy<T> strategy;

public:
    Sorter(vector<T> d, SortStrategy<T> s)
        : data(move(d)), strategy(s) {}

    // 运行时切换策略
    void setStrategy(SortStrategy<T> s) { strategy = s; }

    // 执行排序
    void execute() {
        cout << "排序前: ";
        for (const auto& x : data) cout << x << " ";
        cout << endl;

        strategy(data);

        cout << "排序后: ";
        for (const auto& x : data) cout << x << " ";
        cout << endl;
    }
};

int main() {
    vector<int> nums = {3, -1, 4, -5, 2, -9, 7};

    // 使用升序策略
    cout << "=== 升序 ===" << endl;
    Sorter<int> sorter(nums, ascendingSort<int>);
    sorter.execute();

    // 切换为降序策略
    cout << "\n=== 降序 ===" << endl;
    sorter.setStrategy(descendingSort<int>);
    sorter.execute();

    // 切换为绝对值排序
    cout << "\n=== 绝对值排序 ===" << endl;
    sorter.setStrategy(absoluteSort);
    sorter.execute();

    return 0;
}

语法规则：策略模式结构

template<typename T>
using Strategy = function<void(vector<T>&)>;

class Context {
    Strategy<T> s;
    void setStrategy(Strategy<T> ns) { s = ns; }
    void execute() { s(data); }
};

举一反三：策略模式将算法从使用它的类中分离出来，使其可独立变化。C++ 实现：编译时用模板参数（零开销），运行时用 function/virtual（灵活）。

语法规则：编译时 vs 运行时策略

// 编译时：template <typename Strategy>
// 运行时：function<void(Data&)> strategy;
// 编译时零开销但编译慢，运行时灵活但有间接调用开销

举一反三：模板策略每个实例化生成独立代码（类似 Rust trait 单态化）。function 策略通过类型擦除统一接口。选择取决于是否需要运行时切换。

语法规则：greater 函数对象

#include <functional>
sort(v.begin(), v.end(), greater<int>());
// greater<T> 返回 a > b
// 等价于 Lambda: [](int a,int b){ return a > b; }

举一反三： 预定义了 plus、minus、multiplies、greater、less 等常用函数对象。它们可以像普通函数一样用在 STL 算法中。
