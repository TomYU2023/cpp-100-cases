案例 99：type_traits 类型萃取实战

综合运用 type_traits 实现编译期类型检查、转换和信息查询。

核心知识点：type_traits、类型转换、类型检查、组合萃取

完整代码（含逐行注释）

// 案例 99：type_traits 类型萃取实战
// 编译期类型检查和转换的综合应用
#include <iostream>
#include <type_traits>
#include <string>
#include <vector>
using namespace std;

// 1. 编译期确保类型符合要求
template <typename T>
class NumericArray {
    static_assert(is_arithmetic<T>::value,
                  "NumericArray 只接受算术类型");
    vector<T> data;
public:
    void push(const T& val) { data.push_back(val); }
    size_t size() const { return data.size(); }
};

// 2. 使用类型转换萃取
template <typename T>
auto getDecayedValue(T&& val) {
    using RawType = decay_t<T>;           // 去除引用和 CV 限定
    using SignedType = make_signed_t<RawType>; // 转为有符号
    cout << "原始类型: " << typeid(T).name() << endl;
    cout << "退化类型: " << typeid(RawType).name() << endl;
    cout << "有符号类型: " << typeid(SignedType).name() << endl;
}

// 3. 条件类型选择
template <typename T>
struct StorageType {
    // 小类型直接存值，大类型用指针
    using type = conditional_t<
        (sizeof(T) <= sizeof(void*)),
        T, T*>;
};

int main() {
    // 1. 类型检查
    cout << "=== 类型检查 ===" << endl;
    cout << "int 是算术类型: " << is_arithmetic<int>::value << endl;
    cout << "string 是算术类型: "
         << is_arithmetic<string>::value << endl;
    cout << "int 是整型: " << is_integral<int>::value << endl;

    // 2. 静态断言
    NumericArray<int> intArr;    // OK
    intArr.push(42);
    // NumericArray<string> strArr; // 编译错误！

    // 3. 类型转换萃取
    cout << "\n=== 类型转换 ===" << endl;
    getDecayedValue(42);

    // 4. 条件类型选择
    StorageType<int>::type smallVal = 42;       // 直接存储
    StorageType<array<int, 100>>::type largeVal; // 指针存储
    cout << "\nStorageType<int>: sizeof = "
         << sizeof(smallVal) << " (直接存储)" << endl;
    cout << "StorageType<array<int,100>>: sizeof = "
         << sizeof(largeVal) << " (指针)" << endl;

    return 0;
}

语法规则：type_traits 三大类

// 1. 类型检查: is_integral, is_floating_point, is_pointer
// 2. 类型转换: remove_const, add_pointer, decay, make_signed
// 3. 类型关系: is_same, is_base_of, is_convertible

举一反三：_v 后缀 (C++17) 替代 ::value。_t 后缀 (C++14) 替代 typename ...::type。组合使用实现复杂条件：is_arithmetic_v && !is_const_v。

语法规则：static_assert 编译期断言

template <typename T>
class Container {
    static_assert(is_copy_constructible<T>::value,
        "T 必须支持拷贝构造");
};
// 编译失败时输出自定义错误信息

举一反三：static_assert 是模板代码中的重要工具。在实例化时检查类型约束，比运行时检查和注释都更可靠。C++17 起第二个参数（错误消息）可选。

语法规则：组合萃取技巧

// 确保 T 是整数但非 bool（bool 满足 is_integral）
static_assert(is_integral_v<T> && !is_same_v<T, bool>);
// 移除所有限定后检查
using Base = remove_cvref_t<T>;
// 判断是否可调用
static_assert(is_invocable_v<Func, Args...>);

举一反三：实际项目中很少只用一个萃取。组合判断更精确。注意 is_integral 包含 bool。remove_cvref_t (C++20) 相当于 decay_t 但不退化数组/函数。
