案例 74：状态模式 — TCP 连接状态机

模拟 TCP 连接的 CLOSED/ESTABLISHED 状态流转，展示状态模式如何用多态消除条件分支。

核心知识点：状态模式、有限状态机 FSM、前置声明、策略 vs 状态的区别

完整代码（含逐行注释）

// 案例 74：状态模式 — TCP 连接状态机
// 状态模式：允许对象在内部状态改变时改变其行为，看起来像换了类。
// 本案例用状态机模拟 TCP 连接的状态流转。

#include <iostream>
#include <memory>
#include <string>

/* 语法规则：TCP 连接状态机
   TCP 连接状态：
     CLOSED → [connect] → SYN_SENT → [收到SYN+ACK] → ESTABLISHED
     ESTABLISHED → [close] → FIN_WAIT → CLOSED
   状态模式 vs if-else：用多态替代大量 switch/if，每个状态是一个类。

   举一反三：游戏角色状态（站立/跑步/跳跃/攻击）、
   订单状态机（待支付/已支付/已发货/已完成/已取消）、
   工单流程（新建/处理中/待审核/已关闭）都适合用状态模式。
*/

// 前置声明
class TcpConnection;

// 状态基类
class TcpState {
public:
    virtual ~TcpState() = default;
    virtual std::string name() const = 0;
    virtual void connect(TcpConnection* conn);
    virtual void disconnect(TcpConnection* conn);
    virtual void send(TcpConnection* conn, const std::string& data);
    virtual void receive(TcpConnection* conn, const std::string& data);
};

// TCP 连接（Context）
class TcpConnection {
    std::unique_ptr<TcpState> state;  // 当前状态
public:
    // 初始状态由外部设置
    void setState(std::unique_ptr<TcpState> s) { state = std::move(s); }

    void open() { state->connect(this); }
    void close() { state->disconnect(this); }
    void send(const std::string& data) { state->send(this, data); }
    void receive(const std::string& data) { state->receive(this, data); }

    std::string currentState() const { return state ? state->name() : "NONE"; }
};

// 默认实现：非法操作时的处理
void TcpState::connect(TcpConnection* conn) {
    std::cerr << "[错误] " << name() << " 状态下不能 connect" << std::endl;
}
void TcpState::disconnect(TcpConnection* conn) {
    std::cerr << "[错误] " << name() << " 状态下不能 disconnect" << std::endl;
}
void TcpState::send(TcpConnection* conn, const std::string& data) {
    std::cerr << "[错误] " << name() << " 状态下不能 send" << std::endl;
}
void TcpState::receive(TcpConnection* conn, const std::string& data) {
    std::cerr << "[错误] " << name() << " 状态下不能 receive" << std::endl;
}

// 具体状态 1：已关闭
class ClosedState : public TcpState {
public:
    std::string name() const override { return "CLOSED"; }
    void connect(TcpConnection* conn) override {
        std::cout << "[CLOSED] 发起连接..." << std::endl;
        conn->setState(std::make_unique<class EstablishedState>());
        std::cout << "[→] 状态变为 ESTABLISHED" << std::endl;
    }
};

// 具体状态 2：已建立连接
class EstablishedState : public TcpState {
public:
    std::string name() const override { return "ESTABLISHED"; }
    void disconnect(TcpConnection* conn) override {
        std::cout << "[ESTABLISHED] 关闭连接..." << std::endl;
        conn->setState(std::make_unique<ClosedState>());
        std::cout << "[→] 状态变为 CLOSED" << std::endl;
    }
    void send(TcpConnection* conn, const std::string& data) override {
        std::cout << "[发送] " << data << std::endl;
    }
    void receive(TcpConnection* conn, const std::string& data) override {
        std::cout << "[接收] " << data << std::endl;
    }
};

int main() {
    TcpConnection conn;
    conn.setState(std::make_unique<ClosedState>());
    std::cout << "当前状态: " << conn.currentState() << std::endl;

    // 未连接时不能发送（会报错）
    conn.send("Hello");  // 报错

    // 建立连接
    conn.open();  // CLOSED → ESTABLISHED
    std::cout << "当前状态: " << conn.currentState() << std::endl;

    conn.send("Hello World");
    conn.receive("ACK");

    // 关闭
    conn.close();  // ESTABLISHED → CLOSED
    std::cout << "当前状态: " << conn.currentState() << std::endl;

    return 0;
}

语法规则：前置声明（forward declaration）

class MyClass;  // 仅声明，不定义
// 在需要完整定义前可以使用指针/引用
void foo(MyClass* p);  // OK
// MyClass obj;  // 错误：需要完整定义

💡 举一反三：前置声明解决头文件循环依赖问题。只要不访问成员/不创建对象，指针和引用就足够。减少编译时间，是大型 C++ 项目的必备技巧。
