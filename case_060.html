新语法精讲

Windows 剪贴板 API 流程

语法规则：剪贴板操作的标准流程：(1) OpenClipboard → (2) GetClipboardData/SetClipboardData → (3) CloseClipboard。每次 Open 后必须 Close（类似 RAII），否则其他程序无法使用剪贴板。
// 标准读取流程
OpenClipboard(nullptr);
HANDLE h = GetClipboardData(CF_TEXT);
char* text = (char*)GlobalLock(h);
string result(text);
GlobalUnlock(h);
CloseClipboard();
// 标准设置流程
OpenClipboard(nullptr);
EmptyClipboard();
HGLOBAL hMem = GlobalAlloc(GMEM_MOVEABLE, len + 1);
memcpy(GlobalLock(hMem), data, len + 1);
GlobalUnlock(hMem);
SetClipboardData(CF_TEXT, hMem);
CloseClipboard();

GlobalAlloc / GlobalLock 与 Windows 全局堆

语法规则：GlobalAlloc 在全局堆（Global Heap）中分配内存。这在 32/64 位 Windows 中已经不重要（进程地址空间独立），但剪贴板 API 仍沿用这个传统。SetClipboardData 传入的内存块所有权转移给系统，调用者不能再释放。
HGLOBAL hMem = GlobalAlloc(GMEM_MOVEABLE, size);
// GMEM_MOVEABLE 允许系统移动内存块以优化碎片
// 使用前必须 Lock：
char* p = (char*)GlobalLock(hMem);
strcpy(p, "data");
GlobalUnlock(hMem);
// 注意：SetClipboardData 后不要 GlobalFree！

举一反三

  • 使用 Ditto（开源剪贴板管理器）的架构思想：
    Ditto 使用 SQLite 存储剪贴板历史，支持搜索、同步、加密。其核心思路：
1. clipboard hook（注册剪贴板查看器）获取实时通知
2. SQLite 持久化存储
3. 全局快捷键唤出搜索界面
我们的实现只是轮询版本，没有 hook。

    Ditto 是 Windows 上最流行的剪贴板管理工具，开源（C++），值得阅读源码学习。

  • 跨平台剪贴板支持：
