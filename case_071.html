案例 71：工厂模式 — 对象注册表

实现一个可自动注册的产品工厂，新增 Animal 子类无需修改工厂代码，演示开闭原则和插件式架构。

核心知识点：工厂模式、注册表模式、函数内静态变量、结构化绑定 C++17、匿名命名空间、宏自动注册

完整代码（含逐行注释）

// 案例 71：工厂模式 — 对象注册表
// 工厂模式：定义一个创建对象的接口，让子类决定实例化哪个类。
// 本案例实现一个"自动注册工厂"——新增产品类无需修改工厂代码。

#include <iostream>
#include <string>
#include <memory>
#include <unordered_map>
#include <functional>

/* ── 语法规则：工厂模式家族 ──
   简单工厂：一个工厂类根据参数创建不同产品（需要修改工厂代码）
   工厂方法：每个产品有一个对应的工厂（产品与工厂一对一）
   抽象工厂：创建一族相关产品（如 UI 工厂创建按钮+文本框+窗口）

   本案例展示的是"注册式工厂"——结合了简单工厂的便利性和工厂方法的扩展性。
   新加产品只需注册，完全符合开闭原则（对扩展开放，对修改关闭）。

   示例：
     // 传统方式（需要改工厂代码）
     if (type == "car") return new Car();
     else if (type == "bike") return new Bike();

     // 注册式（不改工厂代码）
     Factory::register("car", []{ return make_unique<Car>(); });
     Factory::register("bike", []{ return make_unique<Bike>(); });
     auto p = Factory::create("car");

   举一反三：几乎所有插件系统都基于注册式工厂。
   游戏引擎的 Component 系统、浏览器的 MIME 处理器都用这种模式。
*/

// 步骤 1：产品基类
class Animal {
public:
    virtual ~Animal() = default;
    virtual void speak() const = 0;
    virtual std::string type() const = 0;
};

// 步骤 2：具体产品
class Dog : public Animal {
public:
    void speak() const override { std::cout << "汪汪！" << std::endl; }
    std::string type() const override { return "Dog"; }
};

class Cat : public Animal {
public:
    void speak() const override { std::cout << "喵喵～" << std::endl; }
    std::string type() const override { return "Cat"; }
};

class Duck : public Animal {
public:
    void speak() const override { std::cout << "嘎嘎！" << std::endl; }
    std::string type() const override { return "Duck"; }
};

// 步骤 3：注册式工厂
class AnimalFactory {
public:
    // 创建器类型：不接受参数，返回 unique_ptr<Animal>
    using Creator = std::function<std::unique_ptr<Animal>()>;

    // 注册产品：绑定类型名和创建函数
    static void registerAnimal(const std::string& name, Creator creator) {
        registry()[name] = std::move(creator);
    }

    // 创建产品：根据名字查找并调用创建器
    static std::unique_ptr<Animal> create(const std::string& name) {
        auto it = registry().find(name);
        if (it == registry().end()) {
            std::cerr << "未知动物类型: " << name << std::endl;
            return nullptr;
        }
        return it->second();  // 调用创建函数
    }

    // 列出所有已注册的产品
    static void listAll() {
        std::cout << "已注册的动物类型:" << std::endl;
        for (const auto& [name, _] : registry()) {
            std::cout << "  - " << name << std::endl;
        }
    }

private:
    // 注册表：类型名 → 创建函数 的映射
    // 用函数内静态变量确保全局唯一且延迟初始化
    static std::unordered_map<std::string, Creator>& registry() {
        static std::unordered_map<std::string, Creator> reg;
        return reg;
    }
};

// 步骤 4：自动注册宏（可选，让注册更优雅）
// 定义一个宏，在全局作用域中自动注册产品
#define REGISTER_ANIMAL(ClassName, DisplayName) \
    namespace {  // 匿名命名空间，限制作用域在本文件 \
        struct AutoReg_##ClassName { \
            AutoReg_##ClassName() { \
                AnimalFactory::registerAnimal(DisplayName, [] { \
                    return std::make_unique<ClassName>(); \
                }); \
            } \
        } autoReg_##ClassName;  // 全局对象，构造时自动注册 \
    }

// 使用宏自动注册（编译时自动执行）
REGISTER_ANIMAL(Dog, "dog")
REGISTER_ANIMAL(Cat, "cat")
REGISTER_ANIMAL(Duck, "duck")

int main() {
    // 手动注册一个（展示不用宏的方式）
    AnimalFactory::registerAnimal("custom", [] {
        // 可以用 lambda 创建匿名子类
        struct CustomAnimal : Animal {
            void speak() const override { std::cout << "我是自定义动物!" << std::endl; }
            std::string type() const override { return "Custom"; }
        };
        return std::make_unique<CustomAnimal>();
    });

    AnimalFactory::listAll();

    // 创建并调用
    std::cout << "\n=== 批量创建 ===" << std::endl;
    auto animals = {
        AnimalFactory::create("dog"),
        AnimalFactory::create("cat"),
        AnimalFactory::create("duck"),
        AnimalFactory::create("custom")
    };

    for (auto& animal : animals) {
        if (animal) {
            std::cout << animal->type() << " says: ";
            animal->speak();
        }
    }

    // 测试不存在的类型
    std::cout << "\n=== 测试错误处理 ===" << std::endl;
    auto unknown = AnimalFactory::create("dragon");

    return 0;
}

语法规则：函数内静态变量（Meyers' Singleton）

MyClass& getInstance() {
  static MyClass instance;  // 第一次调用时初始化，线程安全(C++11+)
  return instance;
}

💡 举一反三：函数内 static 变量是最简单的单例模式实现。C++11 保证其初始化是线程安全的（magic statics）。比传统指针+双重检查锁定简洁太多。

语法规则：结构化绑定 Structured Binding (C++17)

std::map<int, std::string> m = {{1,"a"},{2,"b"}};
for (const auto& [key, value] : m) {  // 直接解包 pair
  std::cout << key << ": " << value << '\n';
}

💡 举一反三：结构化绑定适用于 pair/tuple/数组/简单结构体。C++17 编译选项：-std=c++17。范围 for + 结构化绑定 = 遍历 map 的最优雅写法。

语法规则：匿名命名空间（anonymous namespace）

namespace {  // 没有名字的命名空间
  int hiddenVar = 42;  // 等效于 static int hiddenVar;
}

💡 举一反三：匿名命名空间内的符号仅在当前翻译单元(.cpp文件)可见，替代 C 语言的 static 全局变量。推荐用匿名 namespace 而非 static。
