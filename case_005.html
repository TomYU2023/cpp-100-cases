案例 5：数组去重+排序

对一个整数数组进行去重并排序输出，练习数组操作、STL 容器的基本使用。

核心知识点：vector、sort、unique、迭代器

完整代码（含逐行注释）

// 案例 5：数组去重 + 排序
// 使用 STL 算法高效完成去重和排序
#include <iostream>
#include <vector>    // 动态数组
#include <algorithm> // sort()、unique()
using namespace std;

int main() {
    // 初始化一个包含重复元素的数组
    vector<int> nums = {5, 2, 8, 2, 9, 1, 5, 5, 8, 3};

    cout << "原始数组：";
    for (int n : nums) cout << n << " ";
    cout << endl;

    // 第一步：排序（unique 只能移除相邻的重复元素，所以必须先排序）
    sort(nums.begin(), nums.end());

    // 第二步：去重，unique 返回新逻辑结尾的迭代器
    auto last = unique(nums.begin(), nums.end());

    // 第三步：删除尾部无用元素
    nums.erase(last, nums.end());

    cout << "去重排序后：";
    for (int n : nums) cout << n << " ";
    cout << endl;

    return 0;
}

运行结果

原始数组：5 2 8 2 9 1 5 5 8 3 
去重排序后：1 2 3 5 8 9

vector 动态数组

【规则】std::vector 是 C++ 标准库的动态数组容器，自动管理内存，支持随机访问和尾部动态增长。适用于元素数量不确定、需要频繁在尾部增删的场景。

【原理】vector 在堆上分配一块连续内存，维护三个指针：_start（起始）、_finish（已使用末尾）、_end_of_storage（容量末尾）。当 push_back 导致 size() == capacity() 时触发扩容——通常是 1.5x 或 2x 增长（MSVC 用 1.5x，GCC 用 2x），然后将旧元素拷贝或移动到新内存，释放旧内存。选择 1.5x 可复用已释放的内存块（几何上收敛），而 2x 永远无法复用。

【常见陷阱】1. 在循环中重复调用 v.size() 作为终止条件，且循环体内对 v 执行 push_back——导致死循环（条件永远满足）；2. 保存了 vector 元素的指针/引用/迭代器，随后执行 push_back 触发扩容——原指针全部失效（reallocation）。

#include <vector>
vector<int> v = {1, 2, 3};
v.push_back(4);  // 末尾追加
cout << v.size(); // 获取大小

【变体练习一】用 reserve() 预分配空间后向 vector 添加 10000 个元素，对比不使用 reserve 的构造方式，统计扩容次数和总耗时（用 chrono 计时）。

【变体练习二】实现一个函数 void print_vector_info(const vector& v)，打印 size()、capacity()、sizeof(v) 和元素起始地址，观察 push_back 过程中这些值的变化，验证扩容倍数。

sort + unique 组合去重

【规则】先排序（sort）再配合 unique（将相邻重复元素移到末尾并返回新逻辑结尾），最后 erase 去除尾部无效元素。这是 vector 去重的标准三步法。

【原理】unique 不真正删除元素——它只是把不重复的元素往前搬，返回指向新逻辑结尾的迭代器。尾部残留的是被移动后留下的未定义值。erase 根据这个迭代器清除尾部。这种「重排 + 截断」的两阶段设计遵循了 STL 的分离关注原则：算法只管排列，容器只管删除。

【常见陷阱】1. 忘记先 sort，直接调用 unique——unique 只移除相邻重复，非相邻的重名元素不会被处理；2. 认为 unique 后 vector 的 size() 会自动变小——它只是返回了新的 end 迭代器，容量不变，必须显式 erase。

sort(v.begin(), v.end());
auto last = unique(v.begin(), v.end());
v.erase(last, v.end());

【变体练习一】对随机乱序的整数数组（长度为 100000），用 v.erase(unique(v.begin(),v.end()),v.end()) 的标准三步去重法处理，再用 unordered_set 直接去重后重建 vector，对比两种方案的耗时和内存使用。

【变体练习二】不改变原有顺序地去除重复元素（保留首次出现的位置）：使用 unordered_set 辅助判断是否已见过该元素，配合 remove_if 实现「保持顺序同时高效去重」的效果。

范围 for 循环

【规则】for(auto x : container) 是 C++11 引入的基于范围的循环语法，自动遍历容器所有元素。适用于不需要索引、只关心元素值的简单遍历场景。

【原理】范围 for 底层等价于：auto __begin = begin(container), __end = end(container); for(; __begin != __end; ++__begin) { auto x = *__begin; ... }。关键细节：如果容器在遍历期间被增删元素，__begin 可能失效，导致未定义行为。使用 const auto& 而非 auto 可避免不必要的拷贝，特别是对 string、vector 等大元素。

【常见陷阱】1. 在范围 for 循环体内对容器进行 push_back/erase，导致迭代器失效——编译器不会警告，运行时行为不可预测；2. 使用 auto 而非 const auto& 遍历 string 或大对象，触发隐式拷贝构造，性能损失可能达 10x 以上。

for (int n : nums) {
    cout << n << " ";
}

【变体练习一】分别用范围 for (auto x)、范围 for (const auto& x) 和传统下标循环遍历一个存有 100 万个 string 的 vector，用 chrono 记录耗时，观察引用的性能优势。

【变体练习二】使用范围 for + 结构化绑定遍历 map，直接在循环头写成 for(const auto& [key, value] : myMap)，练习 C++17 的结构化绑定语法。
