案例 92：SFINAE 与 enable_if

使用 enable_if 在编译期根据类型条件选择不同实现，理解替换失败不是错误。

核心知识点：SFINAE、enable_if、类型萃取、条件编译期选择

完整代码（含逐行注释）

// 案例 92：SFINAE 与 enable_if
// 编译期根据类型条件选择函数重载
#include <iostream>
#include <type_traits>
#include <string>
using namespace std;

// 方式 1: enable_if 作为返回值类型
template <typename T>
typename enable_if<is_integral<T>::value, void>::type
process(T value) {
    cout << "处理整数: " << value << " (加 10 = "
         << value + 10 << ")" << endl;
}

template <typename T>
typename enable_if<is_floating_point<T>::value, void>::type
process(T value) {
    cout << "处理浮点数: " << value << " (平方 = "
         << value * value << ")" << endl;
}

// 方式 2: enable_if 作为默认模板参数
template <typename T,
          typename = enable_if_t<is_arithmetic<T>::value>>
void printDouble(T value) {
    cout << "2 * " << value << " = " << value * 2 << endl;
}

// 方式 3: 用 enable_if 控制类模板
template <typename T, typename Enable = void>
class Container {
public:
    static void info() { cout << "通用容器" << endl; }
};

// 对整数类型的特化
template <typename T>
class Container<T, enable_if_t<is_integral<T>::value>> {
public:
    static void info() { cout << "整数容器（有优化）" << endl; }
};

int main() {
    // 1. 根据参数类型自动选择重载
    process(42);          // 整数版本
    process(3.14);        // 浮点版本

    // 2. 仅限算术类型调用
    printDouble(5);       // OK
    printDouble(3.5);     // OK
    // printDouble(string("hi")); // 编译错误！

    // 3. 类模板的 enable_if
    Container<int>::info();          // 整数容器
    Container<double>::info();       // 通用容器
    Container<string>::info();       // 通用容器

    return 0;
}

语法规则：enable_if 基本用法

template <typename T>
typename enable_if<condition, ReturnType>::type
func(T arg);
// 当 condition 为 true 时，函数有效
// 为 false 时，该重载被 SFINAE 移除

举一反三：SFINAE = Substitution Failure Is Not An Error。模板替换失败时不报错，只移除该重载。enable_if 利用这个机制实现条件编译。C++17 用 enable_if_t 简化。

语法规则：enable_if 的三种位置

// 1. 返回值类型
template<T> enable_if_t<c, R> func(T);
// 2. 默认模板参数
template<T, enable_if_t<c, int> = 0> func(T);
// 3. 非类型模板参数

举一反三：三种方式各有优劣。返回值方式清晰但无法用于构造函数（无返回值）。默认参数方式最通用。C++20 的 requires 和 concepts 替代了大部分 enable_if 场景。

语法规则：类型萃取 is_* 家族

is_integral<T>       // 整数类型
is_floating_point<T> // 浮点类型
is_arithmetic<T>     // 算术类型
is_pointer<T>        // 指针类型
is_same<T, U>        // 类型相同
// #include <type_traits>

举一反三：类型萃取是 enable_if 的最佳搭档。_v 后缀 (C++17) 等价于 ::value。组合类型萃取：is_arithmetic && !is_const。自定义萃取可实现更精细的条件控制。
