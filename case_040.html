案例 40：future+async

使用 future 和 async 实现异步任务，获取线程返回值，理解异步编程模型。

核心知识点：async、future、promise、异步编程

完整代码（含逐行注释）

// 案例 40：future + async
// 异步执行任务并获取返回值
#include <iostream>
#include <future>
#include <vector>
#include <numeric>
#include <chrono>
using namespace std;

// 模拟耗时计算
int heavyComputation(int id, int workload) {
    cout << "任务 " << id << " 开始 (workload=" << workload << ")" << endl;

    // 模拟耗时
    this_thread::sleep_for(chrono::milliseconds(workload * 100));

    int result = 0;
    for (int i = 0; i < workload * 1000000; i++) {
        result += i;
    }

    cout << "任务 " << id << " 完成" << endl;
    return result;
}

int main() {
    // 1. async 异步启动任务
    cout << "=== async 异步任务 ===" << endl;
    auto f1 = async(launch::async, heavyComputation, 1, 5);
    auto f2 = async(launch::async, heavyComputation, 2, 3);
    auto f3 = async(launch::async, heavyComputation, 3, 4);

    // 主线程可以做其他事情...

    // get() 获取结果（会阻塞直到任务完成）
    int r1 = f1.get();
    int r2 = f2.get();
    int r3 = f3.get();

    cout << "结果: " << r1 << ", " << r2 << ", " << r3 << endl;

    // 2. async 延迟执行
    cout << "\n=== 延迟执行 (deferred) ===" << endl;
    auto f4 = async(launch::deferred, heavyComputation, 4, 2);
    cout << "任务 4 还没开始执行..." << endl;
    // 只有在调用 get() 时才真正开始执行
    int r4 = f4.get();
    cout << "任务 4 结果: " << r4 << endl;

    // 3. 使用 future 等待多个并行任务
    cout << "\n=== 并行处理向量 ===" << endl;
    vector<int> data(10000000);
    iota(data.begin(), data.end(), 0);

    auto sumFuture = async(launch::async, [&data]() {
        return accumulate(data.begin(), data.begin() + data.size() / 2, 0LL);
    });
    auto sumFuture2 = async(launch::async, [&data]() {
        return accumulate(data.begin() + data.size() / 2, data.end(), 0LL);
    });

    long long total = sumFuture.get() + sumFuture2.get();
    cout << "总和: " << total << endl;

    return 0;
}

语法规则：std::async 启动策略

#include <future>
auto f = async(launch::async, func, args...); // 新线程执行
auto f = async(launch::deferred, func, args...); // 延迟执行
auto f = async(func, args...); // 由实现决定

举一反三：launch::async 确保在新线程中执行。launch::deferred 只在 get()/wait() 时才执行（惰性求值）。默认是 async|deferred 组合。

语法规则：future::get() / wait()

future<int> f = async(task);
f.wait();    // 等待完成，不获取结果
int v = f.get(); // 获取结果（只能调用一次！）

举一反三：get() 会移动结果，再次调用会抛 future_error。shared_future 允许多次获取。f.valid() 检查是否可获取。wait_for/wait_until 支持超时。

语法规则：async 生命周期陷阱

// 小心：async 返回的 future 在析构时会阻塞等待任务完成
// 所以需要有意保存 future 或使用 shared_future

举一反三：std::async 返回的 future 在析构时如果任务未完成会 join。这是一个坑——如果忘了保存 future 变量，主线程会在此处阻塞。
