新语法讲解

async 启动策略

launch::async：必定在新线程执行。launch::deferred：延迟到 get() 调用时在当前线程执行（懒求值）。launch::async | launch::deferred：由实现选择（默认行为，但不可预测）。

举一反三

// 推荐显式指定策略
auto f = async(launch::async, work);    // 明确异步
auto f = async(launch::deferred, work);  // 明确延迟
// 不推荐
auto f = async(work); // 行为不可预测

future / promise 通信模型

promise 是"生产者端"，通过 set_value() 设值；future 是"消费者端"，通过 get() 获取值。两者成对出现，是一个一次性的通信通道。get() 只能调用一次，之后再调用会抛异常。

举一反三

promise p;
future f = p.get_future(); // 绑定
thread t([&](){ p.set_value(42); });
int val = f.get(); // 阻塞直到值到来
// 第二次 f.get() 会抛 future_error

举一反三

shared_future 多次获取

shared_future sf = promise.get_future().share();
// 可被多个线程同时 get()，不会抛异常
