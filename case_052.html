新语法精讲

Header-Only 库的使用方式

语法规则：Header-Only 库的所有代码都在 .h 文件中。使用时：(1) 下载 .h 文件放在项目目录；(2) 在一个 .cpp 文件中 #define STB_IMAGE_IMPLEMENTATION 再 include；(3) 其他文件直接 include 即可。这种方法避免了链接的麻烦。
// ---- image_loader.cpp（唯一的实现文件）----
#define STB_IMAGE_IMPLEMENTATION
#include "stb_image.h"

// ---- 其他文件 ----
#include "stb_image.h"  // 直接使用，无需 DEFINED 宏

malloc/free 与 RAII 的对比

语法规则：stb 库使用 C 风格的 malloc/free 管理内存。在现代 C++ 中应优先使用 unique_ptr 配合自定义删除器来包装 C 内存，确保异常安全。
// C++ 安全的 stbi 封装
using stbi_ptr = unique_ptr;
stbi_ptr img(stbi_load(path, &w, &h, &c, 0), stbi_image_free);
// img 离开作用域时自动调用 stbi_image_free

c_str() 将 std::string 转为 C 字符串

语法规则：c_str() 返回以 null 结尾的不可修改字符数组指针。仅在调用 C API 时使用。注意返回的指针在原始 string 修改或析构后失效—不要保存并长期使用。
string path = "image.png";
FILE* f = fopen(path.c_str(), "rb");  // C API 需要 const char*
// c_str() 返回的是 const char*，不能修改内容
// path.data() 在 C++17 后也可读写（非 const 版本）
